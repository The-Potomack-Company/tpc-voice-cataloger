double-quote characters in output fields instead of being left as literal words.

Purpose: Auctioneers frequently use verbal quote markers when dictating catalog entries (e.g.,
"quote 19th century unquote" should produce "19th century" with real quotation marks). The current
system prompt has vague instructions ("opening quotation mark") and no fallback if the AI ignores
them.

Output: Improved prompt + client-side post-processor ensuring quotes always convert.

  <behavior>
    - "quote 19th century unquote" -> "\"19th century\""
    - "open quote fine condition close quote" -> "\"fine condition\""
    - "open quote hello end quote world" -> "\"hello\" world"
    - "quote red quote" (odd number) -> "\"red\" (closes at end of match if no closing marker)
    - "no quotes here at all" -> "no quotes here at all" (unchanged)
    - Works on all text fields (title, description, condition, transcript)
    - Handles case-insensitive variants: "Quote", "UNQUOTE"
    - Preserves surrounding text: "a quote nice unquote table" -> "a \"nice\" table"
  </behavior>
  <action>
    Create src/utils/spokenPunctuation.ts exporting a function `applySpokenQuotes(text: string): string`.

    The function should:
    1. Use regex to find paired patterns: (quote|open quote) ... (unquote|close quote|end quote)
    2. Replace the opening marker with `"` and the closing marker with `"`
    3. Handle case-insensitive matching
    4. Handle multiple pairs in a single string
    5. Use standard ASCII double-quote character `"` (0x22), not smart/curly quotes

    Pattern approach:
    - Match opening markers: /\b(open\s+quote|quote)\b/gi
    - Match closing markers: /\b(unquote|close\s+quote|end\s+quote)\b/gi
    - Strategy: find opening marker, find next closing marker after it, replace both. Repeat.
    - If an opening marker has no closing marker, leave it unconverted (it's likely the noun "quote").

    Write tests FIRST in src/tests/spoken-punctuation.test.ts using vitest, then implement.
  </action>
