**Change 3: Add merge rules to SYSTEM_PROMPT**

Add the following section to the end of the SYSTEM_PROMPT string (after the existing CRITICAL RULES section, before the closing backtick):

```
MERGE RULES:
When existing field values are provided in the user message, your job is to MERGE new information with existing values:
- Default behavior: APPEND new information to existing field values. For example, if title is "OAK TABLE" and speaker says "add ROBERT", return "OAK TABLE ROBERT".
- If the speaker says "change X to Y", "replace X with Y", or similar edit instructions, modify the existing value accordingly.
- If the speaker says "add X to the title/description", append X to the existing value.
- Only OVERWRITE a field completely if the speaker explicitly asks (e.g., "replace the description with...").
- If a field has no existing value (marked "(empty)"), write the new extracted value directly.
- For transcript: ALWAYS append new speech to the existing transcript, separated by a newline. Never overwrite existing transcript.
- If the audio contains no information relevant to a field, return the existing value unchanged (do NOT return null for fields that already have values).
```

**Change 1: Update all existing mockFrom implementations**

Every test that calls processAudioWithAi currently mocks `from("items")` to handle `update()` and sometimes `select().eq().maybeSingle()` for transcript. Now the pipeline also does a `select(...fields...).eq("id", itemId).maybeSingle()` call BEFORE the Gemini fetch. Update all mockFrom implementations to handle this.

The pattern is: mockFrom needs to return an object that supports both `update()` and `select()`. The `select()` chain returns `.eq()` which returns `.maybeSingle()`. Use a helper function to reduce duplication:

```typescript
// Add this helper at the top of the describe("processAudioWithAi") block:
function createMockFrom(options: {
  updateCalls?: Array<Record<string, unknown>>;
  existingItem?: Record<string, unknown> | null;
}) {
  const { updateCalls = [], existingItem = null } = options;
  return () => ({
    update: (data: Record<string, unknown>) => {
      updateCalls.push(data);
      return { eq: vi.fn().mockResolvedValue({ error: null }) };
    },
    select: () => ({
      eq: () => ({
        single: vi.fn().mockResolvedValue({ data: existingItem, error: null }),
        maybeSingle: vi.fn().mockResolvedValue({ data: existingItem, error: null }),
      }),
    }),
  });
}
```
