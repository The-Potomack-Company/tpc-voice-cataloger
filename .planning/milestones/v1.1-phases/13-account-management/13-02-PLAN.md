Deactivation flow:
1. User clicks "Deactivate" on AccountRow -> sets `deactivateTarget` to that account
2. ConfirmDialog shows with title "Deactivate Account", message "Deactivate {display_name}? They will not be able to log in until reactivated. Their sessions will not be affected.", confirmLabel "Deactivate", cancelLabel "Keep Active", destructive=true
3. On confirm: call `toggleAccountActive(account.id, false)`, optimistically update local state (set is_active=false), set `togglingId` during call
4. On error: revert optimistic update, show inline error below the row

Reactivation flow:
1. User clicks "Reactivate" on AccountRow -> call `toggleAccountActive(account.id, true)` directly (no confirmation per CONTEXT.md)
2. Optimistically update local state (set is_active=true), set `togglingId` during call
3. On error: revert optimistic update, show inline error

**Settings.tsx modification:**

Add an "Admin" section to the Settings page. It must be conditionally rendered only when the current user has admin role. Since the auth store does not store profile data, query the profiles table on mount (same pattern as AdminRouteGuard).

Add this state at the top of SettingsPage:
```typescript
const [isAdmin, setIsAdmin] = useState(false)
useEffect(() => {
  if (!user) return
  supabase
    .from('profiles')
    .select('role')
    .eq('id', user.id)
    .single()
    .then(({ data }) => {
      if (data?.role === 'admin') setIsAdmin(true)
    })
}, [user])
```
