# Milestone v1.2 Requirements -- UI Overhaul

**Goal:** Adopt the unified TPC design system end-to-end. Replace Tailwind 4 `@theme` styling with token-driven primitives, ship a reusable component library, restyle every screen (mockup-faithful where mockups exist; extrapolated for unmocked surfaces), and add motion + WCAG AA polish. Information architecture unchanged.

**Source of truth:** `docs/design-handoff/tpc-unified-tokens.{css,ts}` and `docs/design-handoff/tpc-unified-base.css`.

**REQ-ID format:** `[CATEGORY]-[NN]`.

---

## v1.2 Requirements

### TOKENS -- Design Tokens & Theme

- [ ] **TOKENS-01** -- Tailwind 4 `@theme` color, font, and radius variables are rebuilt from `docs/design-handoff/tpc-unified-tokens.css` and become the single styling source of truth (no hardcoded hex/oklch in component code).
- [ ] **TOKENS-02** -- Dark token set loads automatically when the user's system preference is `prefers-color-scheme: dark` and when the user explicitly selects Dark in Settings.
- [ ] **TOKENS-03** -- User can toggle theme between Light, Dark, and System in Settings; preference persists per-user (Supabase) for signed-in users and falls back to `localStorage` pre-auth.
- [x] **TOKENS-04** -- A static check (lint rule, regex test, or build-time guard) fails the build if any TS/TSX/CSS file outside `docs/design-handoff/` introduces a hardcoded color hex, `oklch(...)`, or font-family literal. *(Phase 22 Plan 04, 2026-04-30 — Vitest filesystem regex sweep at `src/ui/__tests__/no-hardcoded-literals.test.ts`)*

### TYPE -- Self-Hosted Typography

- [ ] **TYPE-01** -- EB Garamond 400 + 400 italic + 500 italic loads via `@fontsource/eb-garamond` (no Google Fonts CDN request in production network panel).
- [ ] **TYPE-02** -- Inter 400 / 500 / 600 loads via `@fontsource/inter`.
- [ ] **TYPE-03** -- IBM Plex Mono 400 / 500 loads via `@fontsource/ibm-plex-mono`.
- [ ] **TYPE-04** -- Display (italic Garamond), UI (Inter), and metadata (Plex Mono) font stacks are exposed as tokens; a `tnum` utility class enables tabular numerics on item numbers, durations, and counts wherever the mockups show monospaced figures.

### LIB -- Reusable Component Library (`src/ui/`)

- [ ] **LIB-01** -- `Button` primitive ships with primary / secondary / ghost / danger variants, sm / md sizes, optional leading icon -- matches `tpc-unified-base.css` button rules.
- [ ] **LIB-02** -- `Badge` primitive ships with default + ok / warn / err / info tones and an optional dot indicator.
- [ ] **LIB-03** -- `Input` primitive consumes the focus-ring rule (accent border + accent-wash glow) and supports leading/trailing slot content.
- [ ] **LIB-04** -- `Card` primitive renders the standard 10 px-radius surface with rule border and supports a flush-padded variant for list rows.
- [ ] **LIB-05** -- `Eyebrow` primitive renders the uppercase 10 px / 0.14 em-tracked label utility used as section labels throughout the mockups.
- [ ] **LIB-06** -- `Bar` (dataviz progress) primitive renders track + fill at the design's 4 px height with semantic color slot (accent / warn / ok / err).
- [ ] **LIB-07** -- Every app screen consumes the LIB primitives for buttons, badges, inputs, cards, eyebrows, and bars -- no ad-hoc inline styles for these element types remain in screen code.

### SCREEN -- Per-Screen Restyle

**Mockup-faithful (reference: `docs/design-handoff/tpc-voice.jsx`):**

- [ ] **SCREEN-01** -- Sessions list matches the Sessions mockup: date-grouped sections, search-with-filter chip, paired Sale (S) / House (H) tiles with `accent-wash` / `sand-wash` backgrounds, mono session IDs, info / warn status badges, active-recording dot, density and typography.
- [ ] **SCREEN-02** -- Recording screen matches the Recording mockup: back / dots header with eyebrow + display title, current-item card with live transcript caret, full-width live waveform (recent bars in accent), tnum elapsed timer, side back + add controls flanking a hero red stop button with accent-halo shadow.
- [ ] **SCREEN-03** -- Review screen matches the Review mockup: sticky header with sync action, three-stat progress strip (Transcribed / AI cataloged / Needs review) with mini bars, vertical item list with status dot (`ok` / `warn` / `err`), mono item numbers, two-line excerpt clamp, tnum duration, status pills, ghost play button.

**Extrapolated (unmocked -- treatment proposed in each phase's discuss step before code):**

- [ ] **SCREEN-04** -- Item detail / edit view restyled: transcript display, editable structured fields (title, description, condition, estimate, category, measurements), photo grid (house mode), re-record action -- all consuming LIB primitives and the new typographic hierarchy.
- [ ] **SCREEN-05** -- Admin accounts screen restyled: list of specialists, create/deactivate actions, role tags.
- [ ] **SCREEN-06** -- Admin assignments view restyled: receipt-list import flow, session creation, specialist assignment.
- [ ] **SCREEN-07** -- Admin review queue restyled: submitted sessions, edit / return-with-notes / export actions, status tracking.
- [ ] **SCREEN-08** -- Login screen restyled: tokens + display typography, no IA change.
- [ ] **SCREEN-09** -- Role-aware walkthrough overlay restyled to match the new visual language; copy and step structure preserved.
- [ ] **SCREEN-10** -- Bottom-tab navigation chrome restyled (icons, active-state color, density) -- tab structure unchanged.

### MOTION -- Motion & Live Visual Feedback

- [ ] **MOTION-01** -- Recording badge pulses (subtle accent-halo expansion) while the active session is recording.
- [ ] **MOTION-02** -- Live waveform renders real audio amplitude (Web Audio API) during recording -- recent bars shown in accent, older bars decay to ink-4 -- not a static decoration.
- [ ] **MOTION-03** -- Top-level route transitions use a consistent <250 ms cross-fade or slide.
- [ ] **MOTION-04** -- All motion respects `prefers-reduced-motion: reduce` -- pulses stop, waveform falls back to a static recording-active glyph, route transitions become instant.

### A11Y -- Accessibility

- [ ] **A11Y-01** -- Every ink-on-bg combination in both light and dark token sets meets WCAG AA contrast (4.5:1 body, 3:1 large text). Verified by an automated test in `src/ui/__tests__/contrast.test.ts` that runs in CI; failures block the build.
- [ ] **A11Y-02** -- Focus rings are visible on every interactive primitive (Button, Input, Badge-as-button, Card-as-link) using the accent border + accent-wash glow rule from the design.
- [ ] **A11Y-03** -- The role-aware walkthrough (existing v1.1 capability) gains a step introducing the theme toggle, with the same per-user persistence model.

## Out of Scope (this milestone)

- **Information architecture changes** -- nav structure stays as-is; only chrome is restyled (SCREEN-10). Reason: explicit user decision -- isolate visual change from behavioral change.
- **Admin / specialist UX divergence** -- both roles get the same shell. Reason: explicit user decision -- defer specialization until field feedback informs it.
- **Feature additions** -- no new functional capabilities; this is purely a visual / structural overhaul. Reason: scope discipline.
- **48 px+ tap-target floor** -- the v1.0 minimum is intentionally relaxed in favor of mockup density. Reason: explicit user decision -- monitor field-use feedback and revisit if outdoor / glove ergonomics regress.
- **Animation library introduction** (Framer Motion, etc.) -- motion is delivered via CSS transitions and Web Audio API only. Reason: bundle-size discipline.
- **Touch-up of the Chrome extension UI** -- extension is a separate codebase and out of this milestone's scope.
