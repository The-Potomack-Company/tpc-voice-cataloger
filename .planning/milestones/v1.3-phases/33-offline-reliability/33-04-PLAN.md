<objective>
REL-4: make `useAudioRecorder.stopRecording()` always settle even when `db.audio.add` rejects, and retain the recorded blob for retry (D-11/D-12). Today the `onstop` catch only `console.error`s (useAudioRecorder.ts:202-204), so `stopResolveRef` never fires and the `Promise<number | undefined>` from `stopRecording` hangs forever. The fix: in the `onstop` catch retry `db.audio.add` up to 2× (IndexedDB failures are usually transient/quota), then ALWAYS settle — `stopResolveRef.current?.(undefined)`, null the ref, set recorder error state, and stash the blob in `recordingStore` for manual re-save. Keep the `Promise<number | undefined>` signature (D-11) so no caller needs try/catch.

Purpose: a rejected IndexedDB add currently hangs the recorder UI indefinitely with no recovery; the blob is lost. Always-settle + a retry buffer eliminates the hang and preserves the recording.

Output: extended `recordingStore.ts` (retry buffer + error), hardened `useAudioRecorder.ts` onstop, recorder tests proving settle-on-reject + retry-success.

Independence: this plan touches NO file shared with 33-00/01/02/03 and is migration-independent. It can run any time after the phase starts; placed in Wave 3 only for clean sequencing.
</objective>
