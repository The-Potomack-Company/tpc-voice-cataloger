## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local shell → GitHub repo secrets | Secret values cross from the operator's machine into GH encrypted secret storage |
| cataloger `.env` → proxy repo secrets | Supabase values copied across repos; must not transit logs in cleartext |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40.1-01 | Information Disclosure | secret values in shell history / step logs | mitigate | Use `gh secret set --body`/stdin; never `echo` the value; verify via `gh secret list` (names only, GH never returns values) |
| T-40.1-02 | Tampering | wrong repo target (cataloger vs proxy) | mitigate | Every `gh secret` call passes explicit `--repo The-Potomack-Company/tpc-ai-proxy` |
| T-40.1-03 | Denial of Service | comma in ALLOWED_ORIGINS value | accept (here) | Comma is safe inside a stored secret; the split risk lives on the env_vars side and is mitigated in plan 02 by quoting |
</threat_model>
