### Deviation (research correction caught at runtime)
RESEARCH Pattern 2 put `allowed_origins: ${{ secrets.ALLOWED_ORIGINS_PROD }}` **inside `strategy.matrix`** ‚Äî GitHub does **not** allow the `secrets` context in the matrix, so the first push (`4ff3cab`) failed at workflow validation (0 jobs, "workflow file issue"). `python3 yaml.safe_load` cannot catch this (it is valid YAML; the violation is a context-availability rule, which only `actionlint` / GitHub validates). Fixed in `24b5645`: the matrix carries a non-secret `leg: prod|dev` discriminator and each step resolves the per-service origin via `${{ matrix.leg == 'prod' && secrets.ALLOWED_ORIGINS_PROD || secrets.ALLOWED_ORIGINS_DEV }}` (evaluated in step context, where `secrets` IS available). The quoted-value form (Pattern 3) was preserved.
