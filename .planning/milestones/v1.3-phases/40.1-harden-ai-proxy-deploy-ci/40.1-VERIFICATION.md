### Implementation Deviation — Not a Gap

The plan's Pattern 2 put `allowed_origins: ${{ secrets.ALLOWED_ORIGINS_PROD }}` inside `strategy.matrix`. GitHub does not permit the `secrets` context in `strategy`/`matrix`, causing the first push (`4ff3cab`) to fail at workflow validation (0 jobs, "workflow file issue"). The executor correctly caught and fixed this: the matrix carries a non-secret `leg: prod|dev` discriminator, and the per-service origin secret is resolved in step context via a ternary expression (`matrix.leg == 'prod' && secrets.ALLOWED_ORIGINS_PROD || secrets.ALLOWED_ORIGINS_DEV`). The quoted-value form (Pattern 3) was preserved in the fix. The intermediate failure run is preserved in history. The final design is more correct than the plan pattern.
