<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web/extension client -> Cloud Run proxy | untrusted Origin header + untrusted Bearer JWT cross here |
| Cloud Run proxy -> Supabase /auth/v1/user | proxy presents the client's bearer + the anon apikey to validate identity |
| Cloud Run proxy -> Gemini upstream | proxy holds the GEMINI_API_KEY secret; never exposed to clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01 | Spoofing | verifyAuth (auth.js) | mitigate | Bearer token validated by /auth/v1/user round-trip; no token / invalid token / round-trip throw all fail-closed to 401 |
| T-40-02 | Elevation | index.js handler order | mitigate | verifyAuth runs before body read + upstream fetch; a request that fails auth never reaches Gemini |
| T-40-03 | Spoofing | isAllowedOrigin (origin.js) | mitigate | suffix match constrained to https:// + tpc- prefix + .vercel.app suffix; bare vercel.app and non-tpc subdomains rejected |
| T-40-04 | Information disclosure | CORS Access-Control-Allow-Origin | mitigate | ACAO echoes only an allowed origin (never `*`); Vary: Origin set; disallowed origins get no CORS headers |
| T-40-05 | Information disclosure | upstream URL logging | accept | existing sanitizeUpstreamUrlForLog redacts key=; no new logging added in this plan |
| T-40-SC | Tampering | npm/node deps in tpc-ai-proxy | accept | no new runtime deps added (verifyAuth uses built-in fetch; CORS is hand-rolled); only existing devDeps |
</threat_model>
