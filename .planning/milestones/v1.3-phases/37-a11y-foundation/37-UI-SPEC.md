# Phase 37 — UI Design Contract

> Visual and interaction contract for the a11y-foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase is additive accessibility plumbing, not new visual language.** It reuses the existing v1.2 TPC design system (tokens, `src/ui/` primitives, focus rings, themes) verbatim. The contract below records the subset of that system this phase touches plus the new interaction contracts for the focus-trap/`<Modal>` primitive, 44px touch targets, icon-button labels, and the overflow-menu delete affordance. It declares **no new colors, fonts, or visual idioms.**

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual, token-driven — no shadcn, no `components.json`) |
| Preset | not applicable |
| Component library | none (hand-rolled `src/ui/` primitives; all modals are `fixed inset-0` Tailwind overlays — no Radix/headlessui) |
| Icon library | local `src/ui/icons.tsx` (`<Icon name=… />`) |
| Font | Inter (UI) / EB Garamond (display) / IBM Plex Mono (metadata) — all self-hosted via `@fontsource`. Phase 37 surfaces use Inter only. |

Source of truth for all tokens: `docs/design-handoff/tpc-unified-tokens.css` + `tpc-unified-base.css`. Theme is applied by wrapping the app shell in `.tpc` (light) / `.tpc.tpc-dark` (dark); never redefine tokens per-component.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gap inside overflow-menu items, focus-ring offset |
| sm | 8px | Overflow-menu item vertical padding, gap between menu rows |
| md | 16px | Modal body padding, overflow-menu inset from row edge |
| lg | 24px | Modal max content gutter / dialog padding |
| xl | 32px | — (available; not specifically required this phase) |
| 2xl | 48px | — |
| 3xl | 64px | — |

**Exceptions (justified):**

- **44px minimum touch target** (D-06, Codex #46). 44 is **not** a multiple of 8 — it is the **WCAG 2.5.5 / Apple HIG minimum interactive-target size** and is therefore a deliberate accessibility exception, not a layout-scale value. Implemented as the Tailwind utility pair `min-h-11 min-w-11` (Tailwind `11` = 2.75rem = 44px) on action/icon buttons. Hit-area is grown via padding, **not** by enlarging the icon glyph (glyphs stay at their existing 16px size where a larger glyph would look wrong). This exception applies only to the interactive bounding box; internal spacing inside the target still uses the 4-multiple scale above.
- **3px focus-ring glow** — inherited verbatim from the v1.2 focus rule (`box-shadow: 0 0 0 3px var(--accent-wash)`). Not a spacing-scale value; it is a fixed token-driven ring width already shipped in `tpc-unified-base.css`. Documented here so the checker does not flag it as an undeclared spacing value.

## Typography

Phase 37 surfaces (`<Modal>` chrome, overflow-menu items, icon-button tooltips) are UI-register only and reuse two of the existing Inter weights. **No display or mono type is introduced by this phase.**

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Menu item / modal body | 13px | 400 | 1.5 |
| Menu item (active) / button label / modal title | 12.5px | 500 | 1.2 |

- **Weights: 2** (400 regular, 500 medium) — both already in the shipped Inter subset. Gate limit is ≤2; satisfied.
- **Sizes: 2** (13px body, 12.5px label) — drawn from the existing v1.2 type ramp (`13px` body, `12.5px` button label per `tpc-unified-base.css`). Gate limit is ≤4; satisfied with headroom.
- Tooltip text reuses the 12.5px / 500 label role; no new type token.

## Color

Inherited verbatim from v1.2 tokens. This phase adds **zero** new color values. The 60/30/10 split below describes the modal/menu surfaces this phase introduces, mapped onto existing tokens.

| Role | Value (token) | Usage |
|------|---------------|-------|
| Dominant (60%) | `--bg` (oklch 0.985 0.003 240 light / 0.19 dark) | Modal body surface, menu panel surface, page behind overlay |
| Secondary (30%) | `--bg-2` / `--bg-3` + `--rule` | Modal overlay scrim, menu hover row (`--bg-2`), 1px rule borders |
| Accent (10%) | `--accent` (teal-blue, oklch 0.58 0.13 225 light) | Visible focus ring (border) + `--accent-wash` glow only |
| Destructive | `--err` / `--err-wash` (oklch 0.55 0.13 28) | Delete menu item label + `ConfirmDialog` danger button only |

**Accent reserved for:** the visible focus ring on trapped/focusable elements (accent border + `--accent-wash` 3px glow, the existing A11Y-02 rule) and primary action buttons inside migrated modals. **Not** applied to overflow-menu rows, tooltips, or the `⋯` trigger glyph at rest (those are `--ink-2` / `--ink-3` neutral ink, accent only on focus). One accent only — gate satisfied.

**Destructive note:** the overflow-menu **Delete** item is the only destructive surface this phase adds; it routes through the existing `ConfirmDialog` (D-04), which already owns the `--err` danger-button treatment. No new destructive styling is authored.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (per modal) | unchanged — each migrated modal keeps its existing confirm label (e.g. ConfirmDialog "Delete", ReturnDialog "Return"). This phase authors no new modal CTAs. |
| Overflow-menu trigger | `aria-label="More actions"` on the `⋯` icon-button (visible tooltip: "More actions") |
| Overflow-menu Delete item | Visible label: **"Delete"**. Routes to existing `ConfirmDialog` — its existing confirmation copy is reused unchanged. |
| Icon-button labels | Every icon-only button gets an `aria-label` (D-07). Reuse the verb-noun pattern phase 36 shipped (`aria-label="Dismiss"` on ErrorToast). Per-button copy is the planner's call; the contract is: imperative verb, no trailing punctuation, matches the visible tooltip string where a tooltip is shown. |
| Tooltip pattern | Native `title`-style tooltip (mechanism is Claude's discretion per D-07); tooltip string MUST equal the button's `aria-label`. Shown only where hover affordance aids sighted users (not on already-labelled text buttons). |
| Empty state | not applicable — phase 37 introduces no new data views or empty states. |
| Error state | not applicable — reuses existing `ErrorToast` (`role="alert"`, `aria-live="assertive"`); phase 37 does not add error surfaces. |
| Destructive confirmation | Delete: reuses the in-place `ConfirmDialog` semantics (now focus-trapped via D-02). Same confirm copy as today's swipe-delete. No new destructive confirmation text. |

## Interaction Contract (phase-specific)

This section is the load-bearing part of the contract for executors. It is non-visual where possible and additive everywhere.

### `<Modal>` primitive + `useFocusTrap` (D-01, D-02)

- Markup: overlay `fixed inset-0 z-50` scrim (`--bg-3` at reduced opacity, matching the existing hand-rolled idiom) + centered panel with `role="dialog"` and `aria-modal="true"`.
- `useFocusTrap` (hand-rolled, zero-dep) handles: **initial focus** into the panel, **Tab / Shift+Tab cycling** confined to focusable descendants, **Escape closes**, and **focus restoration to the trigger** on close.
- Visible focus inside the trap uses the **existing** A11Y-02 focus rule (accent border + 3px `--accent-wash` glow) — no new ring styling.
- Migrate all ~7 dialog sites onto it: `ConfirmDialog`, `ReturnDialog`, `ItemPeekModal`, `PhotoLightbox`, `MigrationSplash`, plus `fixed inset-0` overlays in `SessionDetail` / `NewSession` / `ItemList`.
- **Do NOT trap** transient toasts/indicators: `ErrorToast`, `RecordingToast`, `RecordingIndicator`, `ContinuousModeControlBar`. These remain `role="alert"`/live-region, never `role="dialog"`.

### 44px touch targets (D-06)

- Apply `min-h-11 min-w-11` (44px) to action/icon buttons across the affected surfaces. Grow hit-area with padding; keep glyph sizes as-is.

### Icon-button labels + tooltips (D-07)

- Every icon-only button: `aria-label` always; visible tooltip where hover helps. Tooltip string == `aria-label`. Consistent with ErrorToast's shipped `aria-label="Dismiss"`.

### Overflow-menu delete affordance (D-03, D-04)

- Each `SessionTile` / `SessionCard` / `ItemCard` row gains a `⋯` icon-button (44px, `aria-label="More actions"`, neutral `--ink-2` at rest) opening a small keyboard- and SR-reachable menu containing **Delete**.
- The menu panel reuses the modal/menu surface tokens above (`--bg` panel, `--rule` border, `--bg-2` hover row, `--radius-lg` corner). Menu items: 13px/400 at rest, the focused/active item draws the accent focus ring; the Delete label uses `--err` ink.
- **Delete** invokes the same `onDelete` the existing `SwipeableRow` calls today, through the (now focus-trapped) `ConfirmDialog`. No new delete logic, no new confirmation copy.
- Existing **swipe gesture stays** as a power-user shortcut; the menu is the accessible-equivalent path, not a replacement.
- `prefers-reduced-motion`: any menu open/close transition must respect the existing reduced-motion fallback (MOTION-04) — instant, no animated reveal when reduce is set.
