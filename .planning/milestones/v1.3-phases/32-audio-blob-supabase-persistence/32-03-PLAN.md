enqueueAudioUpload({ dexieAudioId, itemId, sessionId, ext }):
  storagePath = `audio/${sessionId}/${itemId}/${dexieAudioId}.${ext}`
  db.audioUploadQueue.add({ ..., status:'pending', retryCount:0, createdAt:new Date() })
  export { enqueueAudioUpload as enqueue }  (test stubs reference the alias)

processOneUpload(entry):
  mark 'uploading' (lastAttemptAt) → read db.audio.get(entry.dexieAudioId) → if missing mark 'failed' return
  ONE upload: supabase.storage.from('audio').upload(entry.storagePath, audio.blob, { contentType: audio.mimeType, cacheControl:'31536000', upsert:true })  (NO thumbnail — drop photoUploadQueue:77-85)
  metadata upsert (DAT-5): supabase.from('audio').upsert({ item_id: entry.itemId, storage_path: entry.storagePath, mime_type: audio.mimeType, upload_status:'uploaded' }, { onConflict:'storage_path', ignoreDuplicates:true })
  mark 'uploaded'; on error increment retryCount, if >=MAX mark 'failed' else 'pending' + setTimeout(drainAudioQueue, 4^n*1000)
  trackEvent: reuse the analytics envelope (event_type 'audio.uploaded' / 'audio.upload_failed').

drainAudioQueue / retryFailedUploads / _resetDraining: copy photoUploadQueue.ts:145-190 verbatim, swap photoUploadQueue→audioUploadQueue, photos→audio. Keep navigator.onLine pause + Promise.allSettled batching.

extFromMime(mime: string): string — add to src/utils/audio.ts alongside getPreferredMimeType. Map: audio/webm→'webm', audio/mp4→'m4a', audio/ogg→'ogg'; strip ';codecs=...' first; default 'webm'. NEVER hardcode '.opus' (Pitfall 5 / PATTERNS issue 4).

itemId is the Supabase UUID STRING — never the legacy int, never the `as unknown as number` coercion (PATTERNS issue 1; PhotoCapture.tsx:133 "Always use Supabase UUID").
