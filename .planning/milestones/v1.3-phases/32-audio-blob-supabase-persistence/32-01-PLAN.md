## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon-key client → PostgREST → public.audio | Anon key is public (D-003); table RLS via item→session ownership is the only boundary on metadata rows. |
| anon-key client → Storage API → storage.objects (bucket 'audio') | Public anon key; column-scoped storage RLS on path token [2]=sessionId is the only boundary on blobs. |
| pg_cron (server) → purge-audio edge function | Server-to-server; shared cron secret is the only boundary; function is service-role (bypasses RLS) so the secret gate is load-bearing. |
| purge-audio edge function → storage.objects | Service-role; computes its own path set — never trusts caller-supplied paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Information Disclosure | specialist reads another session's audio blob by guessing path | mitigate | Column-scoped `select` storage RLS, `storage.foldername(storage.objects.name)[2]`=sessionId vs created_by/assigned_to (Task 1) |
| T-32-02 | Tampering | specialist uploads/overwrites into another session's path | mitigate | Scoped `insert` (with check) + `update` (using+with check) storage RLS (Task 1) |
| T-32-03 | Tampering / DoS | specialist deletes another session's blob | mitigate | Scoped `delete` storage RLS — own sessions only (Task 1, D-04) |
| T-32-04 | Spoofing / Elevation | unauthenticated/cross-user caller invokes purge-audio to delete arbitrary blobs | mitigate | purge-audio gates on a shared cron secret + computes its own path set (never caller-supplied) (Task 2) |
| T-32-05 | DoS (self-inflicted) | column-scope collision silently denies legitimate owner access | mitigate | Phase-31 column-qualified fix baked into the first audio migration (Task 1) — grep gate asserts it |
| T-32-06 | Information Disclosure / cost | orphaned S3 binaries leak audio after item delete | mitigate | D-04 delete RLS + client storage.remove (plan 05) + purge-audio orphan backstop (Task 2) |
