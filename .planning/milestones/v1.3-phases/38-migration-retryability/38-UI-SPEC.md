# Phase 38 ‚Äî UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** Phase 38 is **mostly backend** (idempotent `migrateToSupabase`, `needsMigration()` correctness, counter split D-09/D-10). The **only UI surface** is SC3: a **persistent, dismissible in-app banner** for the migration `partial` state with a **retry affordance** ‚Äî "N items not yet synced ‚Äî Retry sync". This is a **thin delta on the existing v1.2 design system** and the partial-state work shipped in Phase 36. It introduces **no new visual language**, no new tokens, and no new primitive families. It reuses: the `WarnBanner` primitive + `warn` semantic family (Phase 36), the `PhotoMigrationBanner` persistent-banner precedent, the `MigrationSplash` error-state retry pattern (existing), the 44px touch target + `tpc-btn` focus ring + `aria-label` conventions (Phase 37). The contract's job is to lock the **banner copy, the retry interaction, and the action-button affordance** the planner/executor will add to `WarnBanner`.

This contract is **upward-compatible with Phase 36's** `38`-bound migration banner row: Phase 36 deliberately shipped the partial copy with **no Retry** and noted "Phase 38 adds Try Again." Phase 38 supplies that retry. The `partial` banner copy stays the single source Phase 36 reconciles against.

## Spacing Scale

Inherited from v1.2. The banner reuses the existing `WarnBanner` internal paddings (`px-4 py-3`, `gap-3`); no new spacing tokens introduced this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing (`px-4` banner horizontal) |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks (`min-h-12` retry button height in the splash error path) |
| 3xl | 64px | Page-level spacing |

**Exceptions (justified):**
- **44px (`min-h-11 min-w-11`)** ‚Äî the WCAG 2.5.5 / Phase-37 minimum touch target for the banner's **Retry sync** button and **Dismiss** (`x`) button. Not on the 4-multiple display scale but is the inherited a11y floor established in Phase 37 (`OverflowMenu`, `Modal` close). Required, not decorative.
- **`bottom-24` (96px)** ‚Äî applies only if any retry feedback surfaces through `ErrorToast` (inherited v1.2 toast offset clearing the bottom-tab nav). Not introduced here; noted because a failed retry funnels through `notifyError` (Phase 36 toast). 96px is a multiple of 4 and a multiple of the 48px (2xl) unit.

## Typography

Inherited from v1.2. The banner uses UI body/label scale only ‚Äî **no display type**. **This phase's UI uses exactly two weights.** The banner title is differentiated from the action by its `role="status"` container, the `warn` glyph, and color ‚Äî **not** by a third font weight. (`font-medium`/500 is the only non-400 weight the banner uses; this matches the live `WarnBanner` source which already renders title at `text-sm font-medium`.)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Banner title / message ("N items not yet synced") | 14px (`text-sm`) | 500 medium (Inter) | 1.5 |
| Action label ("Retry sync") | 14px (`text-sm`) | 500 medium (Inter) | 1.2 |
| Count numeral (the "N") | 14px (`text-sm`, optional `tnum`) | 500 medium | 1.5 |
| Display | not used in this phase | ‚Äî | ‚Äî |

**Weight budget: 2 (400 base inherited app-wide + 500 medium used here).** The action label and the title share weight 500; the action is distinguished by the `refresh` icon, an underline or `tpc-btn` ghost treatment, and its interactive affordance ‚Äî never by introducing a 600/semibold. Do **not** add `font-semibold` to this surface.

**Size budget for this phase's surface: 1 (`text-sm`/14px).** The banner is a single-line-scale element; all text is 14px. (The wider v1.2 system has more sizes; this surface deliberately uses one.)

## Color

Inherited semantic tokens from `tpc-unified-tokens.css` (light + dark). The partial-migration banner draws **only** from the `warn` family ‚Äî partial ‚âÝ error (no failure occurred to the user's data; items are merely not-yet-synced and recoverable). **Accent is NOT used on this surface.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` (`oklch(0.985 0.003 240)` light / `oklch(0.19 0.015 255)` dark) | App background behind the banner |
| Secondary (30%) | `--bg-2` / `--bg-3` | Surrounding chrome (nav, layout) |
| Accent (10%) | `--accent` (`oklch(0.58 0.13 225)`) | **NOT used on the partial banner** ‚Äî reserved for primary CTAs, focus rings, live-recording emphasis (v1.2 contract). The retry button's visible focus ring still uses accent (`tpc-btn` ring) ‚Äî that is the inherited focus-ring use, not new accent surface. |
| Warning / partial (the banner) | `--warn` (`oklch(0.62 0.11 75)`) text on `--warn-wash` surface | Banner background + text + icon + the **Retry sync** action ink |
| Error (retry-failed feedback only) | `--err` (`oklch(0.55 0.13 28)`), `--err-wash` | Only if a retry attempt itself fails and routes through the existing `ErrorToast`/`notifyError` (Phase 36). Not a banner surface. |

**Accent reserved for (unchanged from v1.2):** primary CTAs, focus rings, live-recording emphasis. The single accent surface on this banner is the **focus ring** on the Retry button (inherited `tpc-btn` behavior, A11Y-02) ‚Äî this keeps the 10% accent budget intact. The banner body, icon, and Retry action ink are all `warn` family.

**One accent color confirmed:** `--accent` only. Partial = `warn`, retry-failure = `err`. Distinct, learnable, color-not-sole-signal (icon + copy always accompany ‚Äî WCAG 1.4.1).

> **Deliberate divergence from `MigrationSplash`'s error path:** the splash's *at-login* error button is `bg-accent` white-text (existing, untouched ‚Äî D-08). The new *post-dismiss* banner is **not** the splash: it lives in-layout, is lower-salience, and uses the `warn` family with a ghost/underline Retry ‚Äî accent appears only as the focus ring. Keeping the persistent banner off accent-fill prevents it from competing with real primary CTAs and respects the 10% budget.

## Interaction Contract ‚Äî Partial-Migration Banner

| Behavior | Rule | Source |
|----------|------|--------|
| Visibility | Renders **iff** `useDataMigration` reports `partial` (failures > 0 per D-10) **and** the user has not dismissed it this session. Returns `null` otherwise. | CONTEXT D-06, D-07, D-10 |
| Persistence | **Persistent** ‚Äî does NOT auto-dismiss (unlike the splash partial copy, which auto-dismissed in Phase 36 because it had no retry). This banner stays until the user retries successfully or dismisses. | CONTEXT D-07 |
| Dismiss | `x` button (`aria-label="Dismiss"`, 44px target) hides the banner for the session. Dismiss does **not** mark migration complete ‚Äî the banner re-appears next session while `partial` is still true. | CONTEXT D-07; WarnBanner existing |
| Retry action | **"Retry sync"** button re-runs the **whole** migration (`migrateToSupabase`). Idempotency (D-03/D-05) guarantees no duplicate Supabase rows; already-migrated rows are skipped via the `[oldId+type]` reverse lookup. No partial/targeted retry. | CONTEXT D-05, D-07 |
| Success outcome | After a retry where `needsMigration()` returns false (ground-truth, D-09), the banner unmounts (its `partial` source goes false). No success toast required (silent resolution is acceptable; the banner's disappearance is the signal). Optional: route a one-shot `ok`-tone confirmation through existing `notify` ‚Äî planner's discretion. | CONTEXT D-09, D-10 |
| Retry-failure outcome | If the retry run still leaves failures, the banner remains with the updated count. If the retry *throws*, surface the existing Phase-36 retryable `ErrorToast` via `toUserMessage` ‚Äî do not bury the error. | Phase 36 D-06/D-09 |
| In-flight | While a retry is running, the Retry button shows a disabled/busy state (label may read "Retrying‚Ä¶", `aria-busy="true"`); prevents double-submit. Reduced-motion safe (no spinner animation gated behind `prefers-reduced-motion`). | inherited a11y conventions |
| Count semantics | "N" = **failures only** (D-10 `failed` counter). Idempotent-skips (`alreadyMigrated`) are **never** counted in the banner ‚Äî a clean retry must not falsely report "not yet synced." | CONTEXT D-10 |

## Copywriting Contract

Single source of truth for the partial-migration banner copy. Phase 36's `38`-bound banner row reconciles against this. The "N" interpolates the **failed** count (D-10), pluralized.

| Element | Copy |
|---------|------|
| Primary CTA (banner action) | **`Retry sync`** (verb + noun ‚Äî never bare "Retry") |
| Banner title (partial state) | **`{N} item{s} not yet synced`** (e.g. "3 items not yet synced", "1 item not yet synced") |
| Banner body (optional supporting line) | **`Your data is safe ‚Äî retry to finish syncing.`** (reassurance + next step; em-dash + lowercase continuation) |
| In-flight CTA label | `Retrying‚Ä¶` (button `aria-busy="true"`, disabled) |
| Retry-failed (toast, reuses Phase 36) | `Connection problem ‚Äî try again` (network) / `Something went wrong` (unknown) ‚Äî via `toUserMessage` (Phase 36 D-09) |
| Dismiss affordance | icon-only `x`, accessible name `Dismiss` (existing `WarnBanner` `aria-label="Dismiss"`) |
| Empty state heading | N/A ‚Äî no empty state introduced this phase |
| Empty state body | N/A |
| Error state | Covered above (retry-failed ‚Üí existing toast). No console-only failures. |
| Destructive confirmation | N/A ‚Äî retry is **idempotent and non-destructive** (D-03/D-05 guarantee no duplicate rows, no data loss); no confirmation dialog required. |

**Copy principles (enforce in review):**
- Every CTA carries a noun: **`Retry sync`**, not bare `Retry`. (Phase-36 splash's "Retry Migration" already satisfies this; the in-layout banner standardizes on "Retry sync".)
- State what happened to the user's data: "Your data is safe" ‚Äî reassurance is the point (matches Phase 36 partial copy "Your data is safe").
- Pluralize "item/items" on the count.
- Never surface raw Supabase/JSON/stack text (Phase 36 D-09 funnel).
- Keep the banner copy stable ‚Äî Phase 36 consumes it as the single source.

## Accessibility (inherited, must be preserved)

- `WarnBanner` carries `role="status"` (polite live region) ‚Äî **correct** for a partial-state, non-interruptive banner. Keep it. (Do **not** escalate to `role="alert"` ‚Äî partial is not an error.)
- **Retry sync** button: minimum **44√ó44px** target (`min-h-11 min-w-11`, Phase 37 / WCAG 2.5.5), visible focus ring via `tpc-btn` (v1.2 A11Y-02). Accessible name "Retry sync" (text label ‚Äî no icon-only ambiguity even with the `refresh` glyph).
- **Dismiss** (`x`) button: 44px target, icon-only ‚áí **must** carry `aria-label="Dismiss"` (WCAG 4.1.2) ‚Äî already present in `WarnBanner`. Keep it; bump its target to 44px if currently smaller.
- In-flight retry sets `aria-busy="true"` on the button; busy state announced via the existing live region.
- Partial vs. error is conveyed by **icon (`warn`) + copy**, not color alone (WCAG 1.4.1).
- No icon-only action without an accessible name on this surface ‚Äî confirmed (`refresh` pairs with the "Retry sync" text label; `x` carries `aria-label`).
