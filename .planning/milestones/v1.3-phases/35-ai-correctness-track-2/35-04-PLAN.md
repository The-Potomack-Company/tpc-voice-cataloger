<objective>
Two correctness fixes landing in the single-shot pipeline write path:

1. **Confab guard (D-03/D-04):** after Zod parse, if the model returns a null/whitespace transcript, reject the WHOLE response — write no catalog fields, set `ai_status="failed"`. Single-shot ONLY (not continuous — RESEARCH Pitfall 5). Implemented by throwing a tagged `ConfabRejectedError` that the existing catch maps to `failed` (reuses the single failure-write site + fires `ai.processing_failed` for free).

2. **No-clobber on retry (D-05/D-06):** flag a field as user-owned in Dexie `userEditedFields` when the user edits it via `updateItemField`; the retry write-back skips any flagged field. Clear flags on a FRESH (non-retry) success. Fresh-vs-retry is signalled by an explicit new `isRetry` param threaded from retry call sites (O-1 resolution — NOT the `hasExistingData` proxy).

Purpose: stop persisting invented data on empty audio (SC-2) and stop retries clobbering deliberate user edits (SC-3) — without any Supabase schema change, `updated_at`, or `seed` (Phase 39's lane stays untouched).

Output: edits to `src/services/gemini.ts` (guard + filter + isRetry + clear) and `src/db/items.ts` (flag write). Turns SC-2 and SC-3 green.
</objective>
