<decisions>
## Implementation Decisions

### Output format
- Format: `N x N x N in. (N x N x N cm.)` — matches RFC Invaluable catalog convention
- Inches are the input unit — cm is always auto-calculated (multiply by 2.54)
- Cm rounds to one decimal place (91.4, not 91 or 91.44) — drop trailing `.0` (61, not 61.0)
- Inch fractions displayed as common fractions: `.5` → `1/2`, `.25` → `1/4`, `.75` → `3/4` — uncommon fractions stay decimal
- Dimension order is always H x W x D
- 1-3 numbers supported: single (12 in.), two (24 x 36 in.), three (36 x 24 x 18 in.)
- No dimension labels (H/W/D) in the output — order is implicit by convention

### AI extraction
- Gemini returns an array of numbers (e.g., `[36, 24, 18]`) — structured, not verbatim text
- App formats the numbers into the display string with auto cm conversion
- Verbatim speech captured in existing transcript field — no need for raw measurement text
- If auctioneer doesn't mention measurements → null (consistent with AI-03: no hallucinated values)
- If speech is vague ("pretty big", "large") with no actual numbers → null
- Auctioneers speak inches; Gemini extracts as inch values regardless of how spoken (mixed styles: "thirty-six by twenty-four", "thirty-six tall, twenty-four wide")

### Field structure
- Single string field stored in DB (`measurements: string | null`) — same pattern as title/description/condition
- Single EditableField in ItemCard — tap to edit the formatted string as plain text
- Placed after description: Title → Description → **Measurements** → Condition → Estimate → Category
- Re-format on save: when user manually edits, app attempts to re-parse numbers and re-format with cm conversion. If unparseable (e.g., "irregular shape, see photos"), keeps raw text as-is

### Export and extension integration
- Measurements included in JSON export schema
- Chrome extension at `C:\Users\maser\Projects\TPC_AI_Cataloger` fills RFC Invaluable's dimensions field: `#dimetext` / `name="fld3"` (placeholder: "Dimensions or specifications")
- Extension import logic needs updating to fill fld3 alongside fld1 (title) and fld2 (description)

### Claude's Discretion
- Gemini schema design for the measurements array field
- Formatting utility implementation (`formatMeasurements()`)
- Fraction display logic for edge cases beyond 1/4, 1/2, 3/4
- Re-parse logic for manual edits (how tolerant/flexible the parser is)
- Dexie migration version number

</decisions>
