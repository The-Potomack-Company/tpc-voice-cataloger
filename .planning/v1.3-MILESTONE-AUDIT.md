## Headline finding — SEAM-3 (recommend a follow-up phase)

The live single-item AI write-back (`gemini.ts:421`; continuous mode gated off per D-050) bypasses
Phase 39's `preconditionUpdate` and does an unconditional last-writer-wins update — the exact
lost-write class Phase 39 exists to prevent. The continuous path got it right; the single-item path
forked. Mitigations (user-edit provenance skip, claim guard) are partial and miss a concurrent edit
to a not-yet-touched field. Mirror the `geminiContinuous.ts:286-295` reconcile onto the single-item
path in a follow-up phase (carry to next milestone if not closing here).
