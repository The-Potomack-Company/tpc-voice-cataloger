  <action>
    1. Create `src/services/gemini.ts`:

       **blobToBase64(blob: Blob): Promise<string>** — Convert blob to base64 using ArrayBuffer -> Uint8Array -> btoa pattern. Export for testing.

       **processAudioWithAi(audioId: number, itemId: number, itemType: "house" | "sale"): Promise<void>** — Full pipeline:
       - Get the correct Dexie table based on itemType ("house" -> db.houseVisitItems, "sale" -> db.saleItems)
       - Set aiStatus to "processing" on the item record
       - Fetch audio record from db.audio by audioId. Throw if not found.
       - Convert audio blob to base64 via blobToBase64
       - Strip codec parameters from mimeType: `audioRecord.mimeType.split(";")[0]`
       - Build Gemini request payload with:
         - System instruction (SYSTEM_PROMPT constant — see below)
         - Contents array with text prompt + inlineData (base64 audio + baseMimeType)
         - generationConfig with responseMimeType "application/json" and responseSchema from catalogFieldsJsonSchema
       - POST to `import.meta.env.VITE_GEMINI_PROXY_URL` with JSON body `{ model: "gemini-2.5-flash", payload: <the Gemini request> }`
       - Parse response: `data.candidates[0].content.parts[0].text`
       - Validate with `catalogFieldsSchema.safeParse(JSON.parse(text))`
       - On success: write fields to item — use `undefined` (not null) for null fields so Dexie doesn't store them. Category defaults to "furniture" if null: `fields.category ?? "furniture"`. Set aiStatus to "done".
       - On ANY error (catch block): set aiStatus to "failed", set description to "AI processing failed - audio recorded, awaiting manual review"

       **SYSTEM_PROMPT constant:**
       ```
       You are an auction catalog field extractor. You will receive an audio recording of an auctioneer describing an item.

       Your job is to extract the following fields from EXACTLY what the speaker says:
       - title: The item name/type as spoken
       - description: The item description as spoken
       - condition: The condition assessment as spoken
       - estimate: The price estimate as spoken
       - category: The item category as spoken

       CRITICAL RULES:
       1. Use the speaker's EXACT words. Do not rephrase, improve, or formalize.
       2. If a field is not mentioned in the audio, return null for that field.
       3. Do NOT invent or guess values for unmentioned fields.
       4. If the speaker says "oak table, kinda beat up, maybe two hundred", return those exact words in the appropriate fields.
       ```

       Per user decision: Gemini is a PURE FIELD SPLITTER. No ALL CAPS, no "the"-prefixing, no formal language rewriting. TPC formatting is Phase 6.

       - Test all behaviors listed above
