  <action>
Replace the placeholder `src/pages/Login.tsx` with the full implementation. The component must satisfy all tests from Task 1 and match the UI-SPEC exactly.

Component structure:
- State: `email` (string), `password` (string), `error` (string | null), `submitting` (boolean)
- Get `signIn` from `useAuthStore` (selector: `(s) => s.signIn`)
- Get `navigate` from `useNavigate`

Form submit handler (`handleSubmit`):
1. `e.preventDefault()`
2. Set `error` to `null`, `submitting` to `true`
3. Call `const { error } = await signIn(email, password)`
4. If error: set `error` to `error.message`, set `submitting` to `false`
5. If no error: call `navigate('/', { replace: true })`

JSX layout (matching 12-UI-SPEC.md exactly):

```tsx
<div className="flex items-center justify-center h-dvh bg-white dark:bg-gray-900">
  <div className="w-full max-w-sm mx-4">

      TPC Catalog
    </h1>
    <p className="text-sm text-gray-500 dark:text-gray-400 text-center mt-1 mb-8">
      Speech-to-catalog tool for auctioneers
    </p>

    <form onSubmit={handleSubmit}>
      <div className="mb-4">
        <label htmlFor="email" className="block text-sm font-medium text-gray-700 dark:text-gray-300 mb-1">
          Email
        </label>
        <input
          id="email"
          type="email"
          placeholder="you@example.com"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
          required
          autoFocus
          className="w-full min-h-12 px-4 py-3 rounded-lg border border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-800 text-gray-900 dark:text-gray-100 focus:ring-2 focus:ring-accent focus:border-accent outline-none"
        />
      </div>

      <div className="mb-4">
        <label htmlFor="password" className="block text-sm font-medium text-gray-700 dark:text-gray-300 mb-1">

        />
      </div>

      <button
        type="submit"
        disabled={submitting}
        className="w-full min-h-12 rounded-lg bg-accent text-white font-medium mt-6 disabled:opacity-50"

        {submitting ? (

      )}
    </form>
  </div>
</div>
```

Key behavioral requirements:

- Error uses `role="alert"` for screen reader announcement
- Sign In button text is replaced by spinner during submission (not hidden alongside spinner)
- Error clears at the START of the next submit (set `error` to null before calling signIn)
- On success, navigate with `{ replace: true }` so the login page is not in browser history
- Do NOT clear local Dexie data on login (per CONTEXT.md locked decision)
- Display `error.message` from Supabase verbatim -- do NOT differentiate error types (per 12-RESEARCH.md Pitfall 4)
