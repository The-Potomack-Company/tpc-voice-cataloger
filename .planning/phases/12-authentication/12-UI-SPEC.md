## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding (`px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `mt-2`, `py-2`) |
| md | 16px | Default element spacing (`p-4`, `px-4`, `gap-4`) |
| lg | 24px | Section padding (`p-6`, `py-6`, `mb-6`) |
| xl | 32px | Layout gaps (`mb-8`, `px-8` landscape) |
| 2xl | 48px | Major section breaks, tap target height (`min-h-12`) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: Touch targets use `min-h-12` (48px) on all interactive elements per established Phase 1 convention.

```

|    |  Email                           |  |
|    |  [________________________]      |  |
|    |                                  |  |
|    |  Password                        |  |
|    |  [________________________]      |  |
|    |                                  |  |
|    |  [      Sign In          ]       |  |
|    |                                  |  |
|    |  (inline error text here)        |  |

```

**Specifications:**
- Container: `flex items-center justify-center h-dvh bg-white dark:bg-gray-900`
- Card: `w-full max-w-sm mx-4` (no visible card border -- clean flat layout matching app aesthetic)
- App title: `text-2xl font-bold text-gray-900 dark:text-gray-100 text-center`
- Subtitle: `text-sm text-gray-500 dark:text-gray-400 text-center mt-1 mb-8`
- Input labels: `text-sm font-medium text-gray-700 dark:text-gray-300`
- Input fields: `w-full min-h-12 px-4 py-3 rounded-lg border border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-800 text-gray-900 dark:text-gray-100 focus:ring-2 focus:ring-accent focus:border-accent outline-none`
- Gap between fields: `mb-4` (16px)
- Sign In button: `w-full min-h-12 rounded-lg bg-accent text-white font-medium mt-6 disabled:opacity-50`
- Error text: `text-sm text-red-600 dark:text-red-400 mt-3 text-center`
- Spinner (in button): `animate-spin h-5 w-5 border-2 border-white border-t-transparent rounded-full inline-block` replacing button text

## Interaction States

### Login Form

| State | Visual |
|-------|--------|
| Default | Both inputs empty, Sign In button enabled with `bg-accent` |
| Focused input | `ring-2 ring-accent border-accent` on the focused field |
| Submitting | Sign In button shows spinner, text changes from "Sign In" to spinner only, button is `disabled`, `opacity-50` applied |
| Error | Inline red text appears below the Sign In button: `text-sm text-red-600 dark:text-red-400 mt-3 text-center`. Button re-enables. Error clears on next submit attempt. |
| Success | Redirect to `/` -- no success state shown on login page |

### Password Change Form

| State | Visual |
|-------|--------|
| Collapsed | Row shows "Change Password" with right-chevron |
| Expanded | Form reveals with 3 password fields, Cancel and Update Password buttons |
| Validating | "Update Password" button shows spinner, disabled |
| Validation error | Red text below fields: "Passwords do not match" or "Current password is incorrect" |
| Success | Green text: "Password updated successfully". Form collapses after 2 seconds. |
| Error | Red text below fields with Supabase error message |

### Sign Out Flow

| State | Visual |
|-------|--------|
| Tap "Sign Out" | `ConfirmDialog` appears with title "Sign Out", message, and destructive confirm button |
| Confirm | Auth session ends, redirect to `/login` |
| Cancel | Dialog closes, no action |

### ProtectedRoute

| State | Visual |
|-------|--------|
| Loading (`loading === true`) | Full-screen centered spinner (auth loading screen) |
| Authenticated (`session !== null`) | Renders child routes via `<Outlet />` |
| Unauthenticated (`session === null, loading === false`) | Redirects to `/login` with `replace` (no flash) |

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Form inputs | `<label>` elements with `htmlFor` linking to input `id`. Or `aria-label` if visually labeled. |
| Error announcements | Error text uses `role="alert"` so screen readers announce it immediately |
| Button disabled state | `aria-disabled="true"` alongside `disabled` attribute |
| Loading spinner | `aria-label="Loading"` on spinner container, `role="status"` |
| Focus management (login) | Autofocus on email input on mount (`autoFocus`) |
| Focus management (error) | Focus moves to error message after failed submit |
| Touch targets | All buttons and inputs: `min-h-12` (48px) minimum |
| Color contrast | Error red-600 on white exceeds 4.5:1 ratio. Red-400 on gray-900 exceeds 4.5:1 ratio. |
| Password inputs | `type="password"` on all password fields (no show/hide toggle needed for this internal app) |
