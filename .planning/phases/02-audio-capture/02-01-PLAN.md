<task type="auto" tdd="true">
  <name>Task 2: useAudioRecorder hook + recording store</name>
  <files>src/stores/recordingStore.ts, src/hooks/useAudioRecorder.ts, src/tests/audio-recorder.test.ts</files>
  <behavior>
    - Hook starts in idle status with durationMs=0 and error=null
    - startRecording transitions status from idle to recording
    - startRecording calls getUserMedia with { audio: true, video: false }
    - stopRecording transitions status from recording to idle
    - stopRecording writes a blob to db.audio with correct itemId, itemType, mimeType, durationMs, and createdAt
    - stopRecording returns the audio record ID from Dexie
    - stopRecording calls track.stop() on all MediaStream tracks
    - Hook cleanup on unmount stops active recording and releases stream
    - If getUserMedia throws NotAllowedError, status becomes error with message about permission denied
    - If getUserMedia throws NotFoundError, status becomes error with message about no microphone
    - Recording store exposes isRecording, currentDurationMs, lastSavedAudioId, and lastSavedDurationMs
  </behavior>
  <action>
    1. Create src/stores/recordingStore.ts — Zustand store (NOT persisted) with:
       - `isRecording: boolean` (default false)
       - `currentDurationMs: number` (default 0)
       - `lastSavedAudioId: number | null` (default null) — set after successful save, used by toast
       - `lastSavedDurationMs: number` (default 0) — duration of last saved recording for toast display
       - Actions: `setRecording(isRecording)`, `setDuration(ms)`, `setLastSaved(id, durationMs)`, `reset()`

    2. Create src/hooks/useAudioRecorder.ts:
       - State: status ('idle' | 'requesting' | 'recording' | 'error'), durationMs (number), error (string | null)
       - Refs: mediaRecorderRef, streamRef, chunksRef, startTimeRef, timerIntervalRef, detectedMimeTypeRef
       - `startRecording(itemId: number, itemType: 'house' | 'sale')`:
         a. Set status to 'requesting'
         b. Call getUserMedia({ audio: true, video: false }) — on error, map DOMException names to user-friendly messages (NotAllowedError, NotFoundError, NotReadableError)
         c. Call getPreferredMimeType() to detect format
         d. Create MediaRecorder with detected type (pass mimeType in options only if non-empty)
         e. Wire ondataavailable: push event.data to chunks if event.data.size > 0
         f. Wire onstop: assemble Blob, compute duration, write to db.audio.add({ itemId, itemType, blob, mimeType, durationMs, createdAt: new Date() }), update recording store with lastSaved
         g. Call mediaRecorder.start() (no timeslice argument — per research, Safari has issues with timeslice)
         h. Start a 1-second interval timer that updates durationMs state AND recording store
         i. Set status to 'recording', update recording store isRecording=true
       - `stopRecording()`:
         a. Call mediaRecorder.stop()
         b. Stop interval timer
         c. Stop all tracks: stream.getTracks().forEach(t => t.stop())
         d. Set status to 'idle', update recording store isRecording=false
         e. Return a promise that resolves with the audio record ID (set in onstop handler)
       - useEffect cleanup: if recording is active, stop and release everything
       - Return { status, durationMs, error, startRecording, stopRecording }

    3. Create src/tests/audio-recorder.test.ts using renderHook from @testing-library/react:
       - Test all behaviors listed above
       - Use fake-indexeddb (already in setup) to verify Dexie writes
       - Use the MediaRecorder mocks from setup.ts
       - After stopRecording, query db.audio to verify the record was written with correct fields
       - Test unmount cleanup by unmounting the hook while recording is active
  </action>
