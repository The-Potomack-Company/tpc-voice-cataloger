BatchController state recovery pattern (for reference -- DO NOT modify BatchController):
- State saved to chrome.storage BEFORE any page-changing action (clickSave/clickNext)
- `afterSave` flag: true means "just saved, navigate to next"; false means "just navigated, process current"
- State age check: reject state older than 30 seconds
- Tab isolation: store tabId, verify on resume
- ESC key handler for cancellation

**Private state variables:**
- `importStatus` - current status string
- `importData` - full JSON data object
- `importItems` - items array from JSON
- `importMode` - 'sale' or 'house'
- `currentIndex` - which item index we're processing
- `statistics` - {success, skipped, errors}
- `skippedReceipts` - array of receipt numbers not found (sale mode only)
- `isCancelled` - cancellation flag
- `originatingTabId` - tab isolation
- `escHandler` - ESC key listener reference

**Private helpers:**
- `resetState()` - reset all state to defaults
- `getCurrentTabId()` - get tab ID via chrome.runtime.sendMessage (same as batch pattern)
- `updateStatistics(result)` - increment success/skipped/errors counter
- `saveImportState(afterSave = false)` - save full state to StorageHelper including items, currentIndex, statistics, skippedReceipts, tabId, timestamp, afterSave flag, mode
- `clearImportState()` - remove from StorageHelper
- `fillFieldsVerbatim(item)` - get fields via FormController.getFormFields(), write item.title and item.description directly to .value (empty string if undefined), dispatch input events. NO prefix, NO sanitizer. Log the fill action.

**Public API - `startImport(jsonData)`:**
1. Check if already running (return if so)
2. Reset state
3. Set importData = jsonData, importItems = jsonData.items, importMode = jsonData.session.mode
4. Set importStatus = RUNNING
5. Get and store originatingTabId
6. Add ESC key handler (escHandler pattern from batch)
7. Show ProgressBar with item count
8. Route: if mode === 'sale', call processNextSaleItem(); if mode === 'house', call processNextHouseItem()

**Public API - `processNextSaleItem()`:**
1. Check cancelled/completed (currentIndex >= importItems.length -> completeImport)
2. Get current item from importItems[currentIndex]
3. Get receipt number from item.receiptNumber
4. If no receipt number: skip item (updateStatistics('skipped'), add to skippedReceipts with note "no receipt", increment currentIndex, recurse)
5. Navigate to receipt:
   - Find receipt input element using TPC_CONSTANTS.RFC_SELECTORS.RECEIPT_INPUT
   - Set value to receipt number, dispatch input event
   - Trigger navigation (dispatch Enter key event or click submit -- depends on discovery)
   - NOTE: After navigation, page may reload. Save state BEFORE triggering navigation with afterSave = false (we need to verify the receipt after reload)
6. After reload/navigation, verify: compare PageExtractor.extractReceiptNumber() with expected receipt
   - If mismatch: skip item, add receipt to skippedReceipts, increment currentIndex, save state, continue
   - If match: fill fields verbatim, update statistics('success'), increment currentIndex, save state with afterSave = true, call NavigationHelper.clickSave()
   - After save, page reloads -> auto-resume picks up with afterSave = true -> process next sale item

IMPORTANT NOTE ON SALE MODE NAVIGATION: The receipt navigation mechanism is uncertain. The state machine for sale mode must handle the possibility that:
(a) Setting the receipt input and submitting causes a full page reload (like Save/Next) -- state recovery handles this
(b) The receipt input navigates without page reload (SPA-style) -- need to wait for DOM update before reading receipt

Implement approach (a) as primary -- it matches the Save/Next pattern and is safer. The afterSave flag distinguishes "just navigated to receipt, need to verify and fill" from "just saved, need to go to next receipt."

Actually, for sale mode the state flow is more nuanced. Use a `step` field in saved state:
- `step: 'navigate'` -- need to navigate to this receipt number
- `step: 'fill'` -- just navigated, need to verify receipt and fill fields
- `step: 'save'` -- just filled, need to click Save (or just saved, need to advance to next)

**Public API - `processNextHouseItem()`:**
1. Check cancelled/completed (currentIndex >= importItems.length -> completeImport)
2. Get current item from importItems[currentIndex]
3. Fill fields verbatim on current page
4. Update ProgressBar
5. Update statistics('success'), increment currentIndex
6. Save state with afterSave = true
7. Call NavigationHelper.clickSave()
8. Page reloads -> auto-resume picks up with afterSave = true -> call NavigationHelper.clickNext() -> page reloads -> auto-resume processes next house item

**Public API - `cancelImport()`:**
1. Set isCancelled = true, importStatus = CANCELLED
2. Remove ESC handler
3. Clear import state from storage
4. Hide ProgressBar
5. Show cancellation summary modal (same pattern as BatchController.cancelBatch)

**Public API - `completeImport()`:**
1. Set importStatus = COMPLETED
2. Clear import state from storage
3. Remove ESC handler
4. Hide ProgressBar
5. If sale mode AND skippedReceipts.length > 0: show custom modal with skipped receipt list (use ModalComponent.show with HTML content listing each skipped receipt)
6. Else: use ConfirmationModal.showBatchCompletionModal(statistics)

**Auto-resume - `checkAndResumeImport()`:**
1. Check StorageHelper.get(IMPORT_STATE)
2. If no state: return
3. If state age > STATE_MAX_AGE: clear and return
4. If status !== 'running': clear and return
5. Verify tab ID matches (getCurrentTabId vs saved tabId)
6. Restore all state from saved state
7. Re-add ESC handler
8. Show ProgressBar with restored progress
9. Route based on mode and step/afterSave:
   - Sale mode: check step field to determine where to resume
   - House visit mode: if afterSave is true, call NavigationHelper.clickNext() then processNextHouseItem(); otherwise processNextHouseItem()

**ProgressBar updates:**
- Update ProgressBar.show() at each item with { message: 'Importing items...', current: currentIndex, total: importItems.length, statistics, onCancel: () => ImportController.cancelImport(), cancelText: 'Cancel Import' }

**getState(), getStatistics(), isRunning():** Same as skeleton but return real state.
