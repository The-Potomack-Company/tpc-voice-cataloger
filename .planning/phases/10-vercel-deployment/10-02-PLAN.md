  <instructions>

   - Go to Cloudflare dashboard -> Workers & Pages -> tpc-gemini-proxy
   - Copy the production URL (format: `https://tpc-gemini-proxy.<account>.workers.dev`)

2. **Create Vercel project:**
   - Go to https://vercel.com/new
   - Import the `The-Potomack-Company/tpc-app` repository
   - Set project name to `tpc-catalog`
   - Framework preset should auto-detect as Vite

     - Name: `VITE_GEMINI_PROXY_URL`
     - Value: the Cloudflare Worker URL from step 1
     - Environments: Production, Preview, Development (all three)
   - Click Deploy

3. **Verify first deploy:**
   - Wait for build to complete
   - Note the production URL (should be `tpc-catalog.vercel.app`)
   - If `tpc-catalog` was taken, note the actual URL assigned

4. **If the Vercel URL is NOT `tpc-catalog.vercel.app`:**
   - Tell Claude the actual URL so the proxy CORS can be updated
  </instructions>
  <resume-signal>Provide the production URL (e.g., "deployed at tpc-catalog.vercel.app") or describe any issues</resume-signal>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 3: Enable branch protection on main</name>
  <action>User must configure branch protection — Claude cannot access GitHub settings UI for branch protection rules.</action>
  <instructions>
1. **Wait for CI to run at least once** (needed so GitHub knows the check name):
   - Go to the repo Actions tab: https://github.com/The-Potomack-Company/tpc-app/actions
   - Confirm the "CI" workflow has run at least once (from the push that included ci.yml)
   - If it hasn't run yet, create a test PR to trigger it

2. **Enable branch protection:**
   - Go to: https://github.com/The-Potomack-Company/tpc-app/settings/branches
   - Click "Add branch protection rule" (or "Add classic branch protection rule")
   - Branch name pattern: `main`
   - Check: "Require status checks to pass before merging"
   - Search for and select the status check named `ci`
   - Check: "Require branches to be up to date before merging" (optional but recommended)
   - Save changes

3. **Test it works:**
   - Try opening the repo and confirm the protection badge shows on main
  </instructions>
