Fix the export pipeline to avoid the NotAllowedError caused by navigator.share() being called after heavy async data preparation exhausts the browser's ~5s transient user activation window.

1. In `src/utils/export.ts`, rewrite `exportSession` to use a download-first strategy:
   - Always use the anchor-click download approach as the primary method (it does NOT require transient activation).
   - Remove the `navigator.canShare` / `navigator.share` code path entirely. The Web Share API's transient activation requirement makes it unreliable for exports that involve heavy async data prep (N+1 DB queries + base64 encoding of photos and audio). The anchor download works universally on all platforms including mobile.
   - Keep `buildExportData` and `blobToBase64` exactly as they are — no changes needed.
   - The rewritten `exportSession` should:
     ```typescript
     export async function exportSession(sessionId: number): Promise<void> {
       const data = await buildExportData(sessionId);
       const json = JSON.stringify(data, null, 2);
       const blob = new Blob([json], { type: "application/json" });
       const url = URL.createObjectURL(blob);
       const a = document.createElement("a");
       a.href = url;
       a.download = `tpc-session-${sessionId}.json`;
       a.click();
       URL.revokeObjectURL(url);
     }
     ```

2. In `src/pages/SessionDetail.tsx`: No changes needed. The `handleExport` and `handleConfirm` functions call `exportSession` which is now safe regardless of async delay.

3. In `src/tests/export.test.ts`:
   - Remove the test "calls navigator.share when canShare returns true" since we no longer use the Web Share API.
   - Keep the test "falls back to download link when canShare unavailable" but rename it to "downloads JSON file via anchor click" and simplify it (no need to mock canShare/share). The test should verify:
     - `document.createElement('a')` is called
     - The anchor's `.click()` is called
     - The anchor's `.download` contains the session ID
