  <action>
    Create `src/utils/export.ts` with three exported functions:

    1. `blobToBase64(blob: Blob): Promise<string>` — uses FileReader.readAsDataURL, returns promise resolving to data URL string.

    2. `buildExportData(sessionId: number): Promise<ExportSchema>` — queries session from db.sessions.get(sessionId), throws if not found. Determines table by session.mode. Queries items sorted by sortOrder. For each item, queries photos (sorted by sortOrder) and audio. Converts all blobs to base64. Assembles ExportSchema object with `version: 1`, `exportedAt: new Date().toISOString()`, session data (spread without id using destructuring), and items array. The session object should also exclude `deletedAt` if present.

    3. `exportSession(sessionId: number): Promise<void>` — calls buildExportData, JSON.stringify with 2-space indent, creates File object named `tpc-session-{sessionId}.json` with type `application/json`. If `navigator.canShare?.({ files: [file] })` returns true, calls `navigator.share({ files: [file] })`. Otherwise falls back: `URL.createObjectURL(file)`, creates `<a>` element with download attribute, clicks it, revokes URL.

    Write tests FIRST. For FileReader in test environment (jsdom), mock if needed. For navigator.share/canShare, use vi.spyOn mocks.

    Note: The `ExportSchema` `session` field type is `Omit<Session, "id">` — make sure to destructure out both `id` and `deletedAt` from the session record before including in export.
