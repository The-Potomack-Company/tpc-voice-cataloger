## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge py (0.5 = 2px but badge uses px-2 py-0.5) |
| sm | 8px | Compact element spacing, gap-2, mt-2 |
| md | 16px | Default element spacing, p-4, px-4, gap-4 |
| lg | 24px | Section padding, py-6, mb-6 |
| xl | 32px | Layout gaps, mt-8, mb-8 |
| 2xl | 48px | Touch targets (min-h-12 = 48px) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: 48px minimum (`min-h-12`) per existing UX-02 requirement
- Card padding: 16px (`p-4`) consistent with existing `SessionCard` and metadata sections
- Section header bottom margin: 12px (`mb-3`) per existing pattern

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|

| Label | 12px | 600 (semibold) | 1.33 (16px) | `text-xs font-semibold` |

Source: Extracted from existing codebase. SessionCard uses `font-semibold` for card titles, `text-sm` for metadata, `text-xs font-medium` for badge text. Section headers use `text-sm font-semibold uppercase tracking-wider`. Page titles use `text-2xl font-bold`.

### Phase-Specific Typography Rules
- **Specialist group header**: `text-sm font-semibold uppercase tracking-wider text-gray-500 dark:text-gray-400` (matches existing section headers)
- **Assignee name on SessionCard**: `text-xs text-gray-500 dark:text-gray-400` (matches item count metadata row)
- **Status badge text**: `text-xs font-medium` (matches existing badge pattern)
- **Assignee field label on SessionDetail**: `text-sm text-gray-500 dark:text-gray-400` (matches metadata section labels)
- **Assignee field value on SessionDetail**: `text-sm font-medium text-gray-900 dark:text-gray-100` (matches metadata section values)

## Interaction Contracts

### 1. Specialist Dropdown on NewSession (Admin)

| Property | Value |
|----------|-------|
| Element | Native HTML `<select>` (locked decision) |
| Label | "Assign to" |
| Position | After Notes field, before Submit button |
| Required | Yes (admin cannot submit without selection) |
| Default selection | No default — first `<option>` is "Select a specialist..." (disabled placeholder) |
| Data source | `listAccounts()` from `adminApi.ts`, filtered client-side to `is_active === true` |
| Loading state | `<select>` shows "Loading..." as disabled placeholder while accounts fetch |
| Error state | If accounts fail to load, show inline text: "Could not load accounts. Please try again." with no dropdown |

Styling:
```
w-full px-4 py-3 rounded-lg bg-gray-100 dark:bg-gray-800 text-gray-900 dark:text-gray-100
border border-gray-200 dark:border-gray-700 focus:outline-none focus:ring-2 focus:ring-accent min-h-12 mb-6
```
(Matches existing input styling on NewSession form)

### 2. Specialist-Grouped Collapsible Sections (Admin Sessions List)

| Property | Value |
|----------|-------|
| Nesting | Status section (Active/Completed/Archived) > Specialist group sub-headers |
| Header text | `{display_name} ({count})` — e.g., "Sarah (3)" |
| Header style | `text-sm font-semibold text-gray-500 dark:text-gray-400 uppercase tracking-wider` (same as existing section headers) |
| Chevron | Right-pointing SVG, rotates 90 degrees when expanded. `w-4 h-4 text-gray-500 transition-transform` |
| Default state | All specialist groups expanded by default (locked decision) |
| Toggle behavior | Click header to collapse/expand. Same as existing Completed/Archived toggles. |
| Card spacing within group | `space-y-3` (matches existing card list spacing) |
| Group spacing | `mt-4` between specialist groups within a section |
| Empty group | Do not render group header if a specialist has 0 sessions in that section |

### 3. Admin SessionCard Extensions

| Property | Value |
|----------|-------|
| Assignee name position | Below existing metadata row (mode badge + item count + relative time). New line. |
| Assignee name format | `text-xs text-gray-500 dark:text-gray-400` — "Assigned to {name}" |
| Status badge position | Inline with existing badges (mode badge row), after item count |
| Status badge format | `inline-flex items-center text-xs font-medium px-2 py-0.5 rounded-full` with status-specific colors from Color section above |
| Status badge label | Capitalized status: "Active", "Submitted", "Returned", "Exported" |

### 4. Inline Reassignment on SessionDetail (Admin)

| Property | Value |
|----------|-------|
| Position | New row in metadata section (gray box), after "Last updated" row |
| Display mode | Left: "Assigned to" label. Right: assignee display name (tappable). |
| Edit mode | Tapping name replaces it with a native `<select>` dropdown |
| Save trigger | `onChange` — selecting a new value saves immediately. No confirm button. No confirmation dialog. (Locked decision) |
| Cancel trigger | `onBlur` without change, or selecting current assignee |
| Select styling | `text-sm rounded border border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-800 text-gray-900 dark:text-gray-100 px-2 py-1 focus:outline-none focus:ring-2 focus:ring-accent` |
| Hover on display value | `cursor-pointer hover:text-accent` to signal tappability |
| Specialist view | Row is not rendered at all (admin-only) |

### 5. Auto-Assignment for Specialists

| Property | Value |
|----------|-------|
| Trigger | Specialist creates a session via NewSession form |
| Behavior | `assigned_to` is set to `user.id` automatically. No dropdown shown. No UI indication of assignment. |
| Specialist session cards | Identical to current cards — no assignee name, no status badge |

## State Management

### Loading States

| State | Component | Behavior |
|-------|-----------|----------|
| Role loading | Sessions, NewSession, SessionDetail | Show nothing (blank) until role resolves. Prevents flash between specialist/admin views. (Pitfall 4 from RESEARCH.md) |
| Accounts loading (NewSession) | AssigneeSelect | `<select>` shows disabled "Loading..." option |
| Accounts loading (SessionDetail) | Assignee field | Display name shown (from session data). Select only loads when user taps to edit. |
| Sessions loading | Sessions page | Existing loading pattern (from Phase 14 Zustand hooks) |

### Error States

| Error | Component | Behavior |
|-------|-----------|----------|
| Accounts fetch failure | NewSession | Inline error text replaces dropdown: "Could not load accounts. Please try again." Submit button remains disabled. |
| Reassignment failure | SessionDetail | Toast at bottom of screen: "Reassignment failed. Please try again." Revert display name to previous value. |
| Role fetch failure | All | Treat as specialist (restrictive default). Log error to console. |
