Create `.github/workflows/ci.yml` with a single-job CI pipeline. The project is small (303+ tests run in ~7s, build in ~5s), so a single job with sequential steps is faster than parallelizing into separate jobs (avoids ~30s job startup overhead per job).

Create the directory `.github/workflows/` first (it does not exist yet).

File contents for `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'npm'

      - run: npm ci

      - name: Lint
        run: npx eslint .

      - name: Typecheck
        run: npx tsc -b

      - name: Test
        run: npx vitest --run

      - name: Build
        run: npx vite build
```

Key design decisions:
- **Single job named `ci`**: The job name `ci` becomes the status check context name. Branch protection (Plan 03) will reference this exact name.
- **`npm ci`** over `npm install`: Deterministic lockfile-based install, faster in CI, cleans node_modules first.
- **`actions/setup-node@v4` with `cache: 'npm'`**: Built-in npm cache support. No need for separate `actions/cache` step.
- **Node 22**: Per user decision (current stable LTS).
- **Trigger on push to main AND PRs to main**: Catches both direct pushes (before branch protection) and PRs.
- **Scope**: Main app only. The Cloudflare Worker (proxy/) deploys independently via wrangler. The CI workflow does NOT run wrangler commands.

**Rewrite `proxy/src/index.ts`** -- replace the entire file with dynamic CORS:

```typescript
interface Env {
  GEMINI_API_KEY: string;
  ALLOWED_ORIGINS: string;
}

function isAllowedOrigin(origin: string, allowedOrigins: string): boolean {
  const allowed = allowedOrigins.split(',').map(s => s.trim());
  if (allowed.includes(origin)) return true;
  // Suffix match for Vercel preview deploys (*.vercel.app)
  if (origin.endsWith('.vercel.app') && origin.startsWith('https://')) return true;
  return false;
}

function getCorsHeaders(request: Request, env: Env): Record<string, string> {
  const origin = request.headers.get('Origin') || '';
  if (isAllowedOrigin(origin, env.ALLOWED_ORIGINS)) {
    return {
      'Access-Control-Allow-Origin': origin,
      'Access-Control-Allow-Methods': 'POST, OPTIONS',
      'Access-Control-Allow-Headers': 'Content-Type',
      'Vary': 'Origin',
    };
  }
  return {};
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const corsHeaders = getCorsHeaders(request, env);

    if (request.method === 'OPTIONS') {
      return new Response(null, { status: 204, headers: corsHeaders });
    }

    if (request.method !== 'POST') {
      return new Response(JSON.stringify({ error: 'Method not allowed' }), {
        status: 405,
        headers: { ...corsHeaders, 'Content-Type': 'application/json' },
      });
    }

    try {
      const { model, payload } = (await request.json()) as {
        model: string;
        payload: object;
      };

      const geminiUrl = `https://generativelanguage.googleapis.com/v1beta/models/${model}:generateContent?key=${env.GEMINI_API_KEY}`;

      const geminiResponse = await fetch(geminiUrl, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(payload),
      });

      const responseBody = await geminiResponse.text();

      return new Response(responseBody, {
        status: geminiResponse.status,
        headers: { ...corsHeaders, 'Content-Type': 'application/json' },
      });
    } catch {
      return new Response(JSON.stringify({ error: 'Internal server error' }), {
        status: 500,
        headers: { ...corsHeaders, 'Content-Type': 'application/json' },
      });
    }
  },
};
```

Key implementation details:
- **`Vary: Origin`** header is included in ALL CORS responses (REQUIRED when ACAO is not `*` to prevent caching bugs)
- **`isAllowedOrigin`** checks exact match against comma-separated list AND suffix match for `*.vercel.app` preview deploys
- **Empty headers for disallowed origins**: No `Access-Control-Allow-Origin` header at all -- browser enforces the block
- **`ALLOWED_ORIGINS` is a `[vars]` entry** (not a secret) because origin URLs are not sensitive. The existing `GEMINI_API_KEY` correctly uses `wrangler secret put` because it IS sensitive
- **`Env` interface** updated to include `ALLOWED_ORIGINS: string`

After updating the files, the Worker must be redeployed via `npx wrangler deploy` from the `proxy/` directory. This is a human action that will be verified in Plan 03.
