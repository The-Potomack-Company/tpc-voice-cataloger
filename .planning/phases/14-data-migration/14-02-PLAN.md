<objective>
Rewrite the data access layer and service modules to use Supabase as the source of truth instead of Dexie. Sessions.ts and items.ts delegate to the sessionStore. useSessions.ts hooks become Zustand selectors. gemini.ts writes AI results to Supabase. export.ts reads metadata from Supabase and blobs from Dexie via ID mapping. offlineQueue.ts adapts to Supabase item IDs.

Purpose: This is the core data source switch. After this plan, all metadata reads/writes flow through Supabase/Zustand. Components consuming these modules (Plan 04) will not need to know about the change.

Output: Rewritten sessions.ts, items.ts, useSessions.ts, gemini.ts, export.ts, offlineQueue.ts, and updated tests.
</objective>

       ```typescript
       import { useSessionStore } from '../stores/sessionStore';
       import { useAuthStore } from '../stores/authStore';
       import type { Tables } from './database.types';

       type SupabaseSession = Tables<'sessions'>;

       export async function createSession(
         name: string,
         mode: 'house' | 'sale',
         notes?: string,
       ): Promise<string> {
         const userId = useAuthStore.getState().user?.id;
         if (!userId) throw new Error('Not authenticated');
         return useSessionStore.getState().createSession({ name, mode, notes }, userId);
       }

       export async function updateSession(
         id: string,
         changes: Partial<Pick<SupabaseSession, 'name' | 'notes' | 'status'>>,
       ): Promise<void> {
         await useSessionStore.getState().updateSession(id, changes);
       }

       export async function deleteSession(id: string): Promise<void> {
         await useSessionStore.getState().deleteSession(id);
       }

       export async function getSessionById(
         id: string,
       ): Promise<SupabaseSession | undefined> {
         return useSessionStore.getState().sessions.find(s => s.id === id);
       }
       ```
       NOTE: No more softDeleteSession, restoreSession, archiveSession, unarchiveSession -- Postgres has no deletedAt/archivedAt columns. deleteSession is now a hard delete via Supabase (Postgres cascades items). The permanentlyDeleteSession function is removed (deleteSession IS permanent now).
