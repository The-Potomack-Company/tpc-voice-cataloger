**Fix 1 — gemini.ts (line 182):**
Replace `.single()` with `.maybeSingle()` on the transcript read query. Add a null-check
immediately after: if `currentItem` is null (item was deleted), bail out of the function
early — do not attempt to update a deleted item. The early return should happen before
the `supabaseUpdate.transcript = ...` assignment (around line 183-186).

Change:
```
const { data: currentItem } = await supabase
  .from("items")
  .select("transcript")
  .eq("id", itemId)
  .single();
```
To:
```
const { data: currentItem } = await supabase
  .from("items")
  .select("transcript")
  .eq("id", itemId)
  .maybeSingle();
if (!currentItem) return; // Item deleted mid-processing, bail out
```
