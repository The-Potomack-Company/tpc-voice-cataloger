  <name>Task 1: Build SessionDetail page with metadata, items, and lifecycle controls</name>
  <files>src/pages/SessionDetail.tsx, src/App.tsx</files>
  <action>
    **Create `src/pages/SessionDetail.tsx`:**

    1. **Route param:** Use `useParams<{ id: string }>()` to get session ID. Parse to number. If session not found (useSession returns undefined), show "Session not found" with back link.

    2. **Header area:**
       - Back button (arrow left icon) → navigate to "/" (sessions list)
       - Session name as editable heading: tap the name text to toggle into an input field. On blur/enter, call `updateSession(id, { name })`. On escape, cancel edit.
       - Mode badge: "House Visit" or "Sale Cataloging" pill
       - Status badge: "Active" (green) or "Completed" (gray)

    3. **Metadata section:**
       - Item count (from useSessionItemCount)
       - Created date (formatted)
       - Last updated (relative time, same helper as SessionCard)
       - Notes: editable textarea. On blur, auto-save via `updateSession(id, { notes })`. Per user decision: notes editable anytime from session detail screen. Silent save (no indicator).

    4. **Item list section:**
       - Heading: "Items ({count})"
       - For now (Phase 3), show a placeholder message: "Items will appear here as you record them" if count is 0
       - If items exist (from Phase 2 demo data or future phases), list them with basic info (sortOrder, createdAt). Use `useLiveQuery` to query houseVisitItems or saleItems by sessionId based on session mode.
       - This list will be enhanced in Phase 4 with recording controls.

    5. **Interrupted recording indicator:**
       - Check `uiStore.recordingSessionId === session.id` on mount
       - If true, show a warning banner at top: "A recording may have been interrupted. Check your items for missing audio." with a dismiss button that clears `recordingSessionId` via `setRecordingSession(null)`.

    6. **Action buttons at bottom:**
       - "Mark Complete" button (if status is "active"): calls `updateSession(id, { status: "completed" })` after ConfirmDialog "Mark this session as complete?"
       - "Reopen Session" button (if status is "completed"): calls `updateSession(id, { status: "active" })` after ConfirmDialog "This session was marked complete. Reopen it?"
       - "Delete Session" button (destructive, always visible): calls `softDeleteSession(id)` after ConfirmDialog, then navigates back to "/"

    7. **Styling:** portrait:px-4, landscape:px-8, landscape:max-w-3xl, dark mode. All buttons min-h-12. Sticky header with back button if content is long.

    **Update `src/App.tsx`:** If Plan 02 created a placeholder SessionDetailPage, replace the placeholder import with the real SessionDetailPage import. Ensure route `path="session/:id"` points to the real component.
