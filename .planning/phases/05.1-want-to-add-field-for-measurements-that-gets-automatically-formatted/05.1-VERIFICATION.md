### Anti-Patterns Found

None. All `placeholder` matches in `ItemCard.tsx` are legitimate HTML input placeholder attributes for `EditableField` components — not implementation stubs.

**Note on `inchesToCm` ternary (line 17):** Both branches of `return cm % 1 === 0 ? String(cm) : String(cm)` are identical, making the ternary a no-op. However, this is benign — JavaScript's `String()` on a floating-point number natively drops trailing `.0` (e.g., `String(61.0)` produces `"61"`). The trailing-zero-drop behavior is correct and all 20 tests pass. This is a code clarity issue only, not a functional defect.
