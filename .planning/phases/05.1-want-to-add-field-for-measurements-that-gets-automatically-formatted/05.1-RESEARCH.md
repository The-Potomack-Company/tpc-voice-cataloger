## Architecture Patterns

### Pattern 1: Formatter Utility (Pure Function)
**What:** A pure function that takes an array of numbers (inches) and returns a formatted string with cm conversion.
**When to use:** Called after Gemini returns structured data, and on EditableField save for re-formatting.
**Follows:** Same pattern as `formatEstimate()` in `src/utils/formatEstimate.ts` and `mapCategoryToCode()`.

```typescript
// src/utils/formatMeasurements.ts
// Input: array of 1-3 numbers (inches)
// Output: "36 x 24 x 18 in. (91.4 x 61 x 45.7 cm.)"

export function formatMeasurements(inches: number[]): string {
  if (inches.length === 0) return "";

  const formatInch = (n: number): string => {
    const frac = n % 1;
    const whole = Math.floor(n);
    // Common fractions
    if (Math.abs(frac - 0.25) < 0.001) return whole ? `${whole} 1/4` : "1/4";
    if (Math.abs(frac - 0.5) < 0.001) return whole ? `${whole} 1/2` : "1/2";
    if (Math.abs(frac - 0.75) < 0.001) return whole ? `${whole} 3/4` : "3/4";
    if (frac === 0) return `${whole}`;
    return `${n}`; // Uncommon fractions stay decimal
  };

  const formatCm = (n: number): string => {
    const cm = n * 2.54;
    const rounded = Math.round(cm * 10) / 10;
    // Drop trailing .0
    return rounded % 1 === 0 ? `${Math.round(rounded)}` : `${rounded}`;
  };

  const inchParts = inches.map(formatInch).join(" x ");
  const cmParts = inches.map(formatCm).join(" x ");

  return `${inchParts} in. (${cmParts} cm.)`;
}
```

### Pattern 2: Re-parse on Manual Edit
**What:** When user edits the measurements field manually, try to extract numbers and re-format. If parsing fails, keep raw text.
**When to use:** In the `onSave` handler for the measurements EditableField.

```typescript
// src/utils/formatMeasurements.ts
export function parseMeasurements(raw: string): number[] | null {
  // Try to extract numbers from strings like "36 x 24 x 18" or "36x24x18"
  // Strip "in.", "cm.", parenthetical cm section
  const cleaned = raw.replace(/\(.*\)/g, "").replace(/in\./gi, "").trim();
  const parts = cleaned.split(/\s*x\s*/i);
  const numbers = parts.map(p => {
    // Handle fractions: "36 1/2" -> 36.5
    const fracMatch = p.trim().match(/^(\d+)?\s*(\d+)\/(\d+)$/);
    if (fracMatch) {
      const whole = fracMatch[1] ? parseInt(fracMatch[1]) : 0;
      return whole + parseInt(fracMatch[2]) / parseInt(fracMatch[3]);
    }
    return parseFloat(p.trim());
  });
  if (numbers.some(isNaN) || numbers.length === 0 || numbers.length > 3) return null;
  return numbers;
}

export function reformatMeasurements(raw: string): string {
  const numbers = parseMeasurements(raw);
  if (!numbers) return raw; // Unparseable -- keep as-is
  return formatMeasurements(numbers);
}
```

### Anti-Patterns to Avoid
- **Storing raw array in DB:** Store the formatted string, not the array. The formatted string is what users edit and what gets exported. Re-parsing extracts numbers when needed.
- **Adding measurements to Dexie indexes:** This field is never queried by value -- no index needed. Adding unnecessary indexes bloats storage.
- **Modifying EditableField component:** The existing EditableField handles this fine. Apply re-formatting in the `onSave` callback, not inside the component.

## Common Pitfalls

### Pitfall 1: Floating Point in cm Conversion
**What goes wrong:** `36 * 2.54 = 91.44000000000001` due to IEEE 754
**Why it happens:** JavaScript floating point arithmetic
**How to avoid:** Use `Math.round(cm * 10) / 10` for one-decimal rounding, then check `rounded % 1 === 0` to drop `.0`
**Warning signs:** cm values showing many decimal places

### Pitfall 2: Fraction Display Edge Cases
**What goes wrong:** Values like `.33333` (1/3) or `.125` (1/8) need a decision
**Why it happens:** User decision specifies only 1/4, 1/2, 3/4 as common fractions
**How to avoid:** Only convert exact matches for .25, .5, .75. Everything else stays as decimal number.
**Warning signs:** Fractions like "1/3" or "1/8" appearing unexpectedly

### Pitfall 3: Re-parse Tolerance
**What goes wrong:** User types "irregular shape, see photos" and re-parse tries to extract numbers
**Why it happens:** Parser is too aggressive
**How to avoid:** `parseMeasurements()` returns null if no valid number pattern found. `reformatMeasurements()` returns raw text unchanged when parsing fails.
**Warning signs:** Non-dimensional text getting corrupted on save

### Pitfall 4: Gemini Schema Must Strip $schema and additionalProperties
**What goes wrong:** Gemini API rejects the JSON Schema if it contains `$schema` or `additionalProperties` keys
**Why it happens:** Zod v4's `toJSONSchema()` includes these by default
**How to avoid:** Already handled in `gemini.ts` line 99-101 -- the destructuring strips these. Adding the measurements array field to the schema won't change this. Just ensure the array/number types Zod generates are valid for Gemini.
**Warning signs:** Gemini API 400 errors

### Pitfall 5: Dexie Version Must Be Next Sequential
**What goes wrong:** Dexie throws if version numbers skip or duplicate
**Why it happens:** Current latest is v4; next must be v5
**How to avoid:** Use `db.version(5)` -- verify no other pending migration in another branch
**Warning signs:** "VersionError" on app load

### Pitfall 6: Extension Field Selector Verification
**What goes wrong:** `#dimetext` selector might not exist or might have changed
**Why it happens:** RFC Invaluable DOM can change without notice
**How to avoid:** The extension already uses `setFieldIfPresent()` which guards against null elements. Won't crash, will just silently skip. Can be verified against live site.
**Warning signs:** Measurements not appearing in RFC after import -- check browser console for logs

## Code Examples

### Complete formatMeasurements Utility
```typescript
// src/utils/formatMeasurements.ts

/**
 * Format inch values as common fractions where applicable.
 * Only .25 -> 1/4, .5 -> 1/2, .75 -> 3/4 are converted.
 * All other fractional values stay as decimals.
 */
function formatInchValue(n: number): string {
  const whole = Math.floor(n);
  const frac = n - whole;

  if (Math.abs(frac) < 0.001) return `${whole}`;
  if (Math.abs(frac - 0.25) < 0.001) return whole ? `${whole} 1/4` : "1/4";
  if (Math.abs(frac - 0.5) < 0.001) return whole ? `${whole} 1/2` : "1/2";
  if (Math.abs(frac - 0.75) < 0.001) return whole ? `${whole} 3/4` : "3/4";
  return `${n}`;
}

/**
 * Convert inches to cm, round to 1 decimal, drop trailing .0
 */
function inchToCm(n: number): string {
  const cm = n * 2.54;
  const rounded = Math.round(cm * 10) / 10;
  return rounded % 1 === 0 ? `${Math.round(rounded)}` : `${rounded}`;
}

/**
 * Format an array of 1-3 inch measurements into auction catalog format.
 * Example: [36, 24, 18] -> "36 x 24 x 18 in. (91.4 x 61 x 45.7 cm.)"
 */
export function formatMeasurements(inches: number[]): string {
  if (inches.length === 0) return "";
  const inchStr = inches.map(formatInchValue).join(" x ");
  const cmStr = inches.map(inchToCm).join(" x ");
  return `${inchStr} in. (${cmStr} cm.)`;
}

/**
 * Try to parse a measurement string back into inch values.
 * Handles: "36 x 24 x 18", "36 x 24 x 18 in. (91.4 x 61 x 45.7 cm.)",
 * "12 1/2 x 8", plain numbers.
 * Returns null if unparseable.
 */
export function parseMeasurements(raw: string): number[] | null {
  // Remove parenthetical cm section
  let cleaned = raw.replace(/\(.*?\)/g, "").trim();
  // Remove "in." suffix
  cleaned = cleaned.replace(/\bin\.?\b/gi, "").trim();
  if (!cleaned) return null;

  const parts = cleaned.split(/\s*x\s*/i);
  if (parts.length === 0 || parts.length > 3) return null;

  const numbers: number[] = [];
  for (const part of parts) {
    const trimmed = part.trim();
    // Try fraction: "36 1/2" or "1/4"
    const fracMatch = trimmed.match(/^(\d+)?\s*(\d+)\/(\d+)$/);
    if (fracMatch) {
      const whole = fracMatch[1] ? parseInt(fracMatch[1]) : 0;
      const num = parseInt(fracMatch[2]);
      const den = parseInt(fracMatch[3]);
      if (den === 0) return null;
      numbers.push(whole + num / den);
      continue;
    }
    // Try plain number
    const n = parseFloat(trimmed);
    if (isNaN(n) || n <= 0) return null;
    numbers.push(n);
  }

  return numbers.length > 0 ? numbers : null;
}

/**
 * Re-format a manually edited measurement string.
 * If parseable, returns formatted string with cm conversion.
 * If unparseable (e.g., "irregular shape"), returns raw text unchanged.
 */
export function reformatMeasurements(raw: string): string {
  const trimmed = raw.trim();
  if (!trimmed) return trimmed;
  const numbers = parseMeasurements(trimmed);
  if (!numbers) return trimmed;
  return formatMeasurements(numbers);
}
```
