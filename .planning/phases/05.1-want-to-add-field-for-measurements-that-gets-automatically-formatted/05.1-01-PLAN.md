<feature>
  <name>Measurements formatting utility + DB foundation</name>
  <files>src/utils/formatMeasurements.ts, src/tests/formatMeasurements.test.ts, src/db/types.ts, src/db/index.ts</files>
  <behavior>
    formatMeasurements:
    - [36, 24, 18] -> "36 x 24 x 18 in. (91.4 x 61 x 45.7 cm.)"
    - [12] -> "12 in. (30.5 cm.)"
    - [24, 36] -> "24 x 36 in. (61 x 91.4 cm.)"
    - [36.5] -> "36 1/2 in. (92.7 cm.)"
    - [12.25] -> "12 1/4 in. (31.1 cm.)"
    - [8.75] -> "8 3/4 in. (22.2 cm.)"
    - [10.33] -> "10.33 in. (26.2 cm.)" (uncommon fraction stays decimal)
    - [] -> ""
    - cm 24 inches -> 61 cm (not 61.0 — drop trailing .0)
    - cm 36 inches -> 91.4 cm (one decimal)

    parseMeasurements:
    - "36 x 24 x 18" -> [36, 24, 18]
    - "36 x 24 x 18 in. (91.4 x 61 x 45.7 cm.)" -> [36, 24, 18]
    - "12 1/2 x 8" -> [12.5, 8]
    - "1/4" -> [0.25]
    - "irregular shape, see photos" -> null
    - "" -> null
    - "36 x 24 x 18 x 12" -> null (more than 3)

    reformatMeasurements:
    - "36 x 24" -> "36 x 24 in. (91.4 x 61 cm.)"
    - "irregular shape" -> "irregular shape" (kept as-is)
    - "" -> ""
  </behavior>
  <implementation>
    1. Create src/utils/formatMeasurements.ts with three exported functions:
       - formatMeasurements(inches: number[]): string
       - parseMeasurements(raw: string): number[] | null
       - reformatMeasurements(raw: string): string

    2. Formatting rules (per user decisions):
       - Inch fractions: only .25->"1/4", .5->"1/2", .75->"3/4" are converted; all others stay decimal
       - Cm conversion: multiply by 2.54, round to 1 decimal via Math.round(cm*10)/10, drop trailing .0 via modulo check
       - Output format: "{inches} in. ({cm} cm.)" with " x " separator

    3. Parsing rules:
       - Strip parenthetical cm section, strip "in." suffix
       - Split on " x " (case-insensitive)
       - Handle fraction notation "N M/D" -> N + M/D
       - Return null if any part is NaN, count is 0 or >3, or input is empty

    4. Add measurements?: string to HouseVisitItem, SaleItem, and ExportSchema items array
    5. Add Dexie v5 migration (same indexes as v4 — measurements is not indexed)
  </implementation>
</feature>
