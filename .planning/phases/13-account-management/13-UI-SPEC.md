## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `mt-1`) | Badge gaps, inline vertical nudges |
| sm | 8px (`gap-2`, `py-2`, `px-2`) | Compact element spacing, badge padding |
| md | 16px (`p-4`, `px-4`) | Default card padding, page horizontal padding (portrait) |
| lg | 24px (`gap-6`, `py-6`, `mb-6`) | Page vertical padding, section bottom margin |
| xl | 32px (`mb-8`, `px-8`) | Section breaks, landscape horizontal padding |
| 2xl | 48px (`min-h-12`) | Touch target minimum height |
| 3xl | 64px | Not used in this phase |

Exceptions:
- `gap-3` (12px) is used between action buttons and between row content and action area -- established project pattern, retained here.
- `min-h-12` (48px) on all interactive elements for thumb-zone accessibility (UX-02 requirement).

Source: Existing `Settings.tsx`, `SessionCard.tsx`, `AppLayout.tsx` patterns.

## Component Inventory

### 1. Account Management Page (`AccountManagement.tsx`)

**Layout:** Same page shell as `Settings.tsx` -- `portrait:px-4 landscape:px-8 landscape:max-w-3xl landscape:mx-auto py-6`.

**Structure:**
```
Page Title: "Account Management"
  Back navigation (arrow + "Settings" link)
  "+ Add Specialist" button (top of list, accent text)
  [Expandable creation form -- shown/hidden on button toggle]
  Account list (space-y-3 stack of AccountRow components)
  [Empty state -- shown when no accounts]
```

**Back navigation:** Left-aligned row above the title. Arrow icon (Heroicons `chevron-left`, 20px) + "Settings" text link. Class: `text-sm text-accent font-medium flex items-center gap-1 mb-4`. This is a dedicated page at `/admin/accounts`, so a back link to `/settings` is needed.

### 2. Add Specialist Button

**Appearance:** Full-width button at the top of the account list area.
```
Classes: min-h-12 w-full rounded-lg border-2 border-dashed border-gray-300 dark:border-gray-600
         text-accent text-sm font-medium
         hover:border-accent hover:bg-accent/5 dark:hover:border-accent dark:hover:bg-accent/5
         transition-colors flex items-center justify-center gap-2
```
**Label:** "+ Add Specialist"
**Icon:** Plus icon (Heroicons `plus`, 20px) before text.
**Behavior:** Toggles the expandable creation form. When form is open, button changes to "Cancel" with muted styling (`text-gray-500 border-gray-300`).

### 3. Inline Expandable Creation Form

**Container:** Appears directly below the Add Specialist button when expanded. Uses `bg-gray-50 dark:bg-gray-800 rounded-lg p-4` card style (same as account rows).

**Fields:**

| Field | Type | Placeholder | Validation |
|-------|------|-------------|------------|
| Display name | text input | "Full name" | Required, min 1 char after trim |
| Email | email input | "email@example.com" | Required, valid email format |
| Password | password input | "Temporary password" | Required, min 6 chars (Supabase default minimum) |

**Input styling:** `w-full rounded-lg border border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-700 px-3 py-2 text-base text-gray-900 dark:text-gray-100 placeholder:text-gray-400 dark:placeholder:text-gray-500 focus:outline-none focus:ring-2 focus:ring-accent focus:border-transparent min-h-12`

Note: `text-base` (16px) on inputs prevents iOS Safari auto-zoom on focus.

**Field labels:** `text-sm font-medium text-gray-700 dark:text-gray-300 mb-1` above each input.

**Field spacing:** `space-y-4` between field groups (label + input).

**Submit button:**
```
Classes: w-full min-h-12 rounded-lg bg-accent text-white font-medium text-sm
         hover:bg-accent-hover transition-colors
         disabled:opacity-50 disabled:cursor-not-allowed
Label: "Create Account"
```
**Loading state:** Button shows "Creating..." with `disabled` attribute. Spinner not required -- text change is sufficient given the fast edge function response.

**Error display:** Red text below the form, above the submit button. `text-sm text-red-600 dark:text-red-400 mt-2`. Shows API error message (e.g., "Email already registered", "Password must be at least 6 characters").

**Success behavior:** Form collapses, fields reset, new account appears in list. No toast -- the new row appearing is sufficient feedback.

### 4. Account Row (`AccountRow.tsx`)

**Container:** `bg-gray-50 dark:bg-gray-800 rounded-lg p-4 flex items-center justify-between gap-3`

**Layout:**
```
[Left side: flex-1 min-w-0]
  Display name (primary text, truncate)
  Email (secondary text, truncate, mt-0.5)
  Badge row (flex items-center gap-2 mt-1)
    [Role badge pill]
    [Status badge pill]

[Right side: shrink-0]
  [Deactivate/Reactivate button OR nothing for own admin row]
```

**Role badge -- Admin:** `inline-flex items-center text-xs font-medium px-2 py-0.5 rounded-full bg-blue-100 dark:bg-blue-900/30 text-blue-700 dark:text-blue-300`
Label: "Admin"

**Role badge -- Specialist:** `inline-flex items-center text-xs font-medium px-2 py-0.5 rounded-full bg-indigo-100 dark:bg-indigo-900/30 text-indigo-700 dark:text-indigo-300`
Label: "Specialist"

**Status badge -- Active:** `inline-flex items-center text-xs font-medium px-2 py-0.5 rounded-full bg-green-100 dark:bg-green-900/30 text-green-700 dark:text-green-300`
Label: "Active"

**Status badge -- Deactivated:** `inline-flex items-center text-xs font-medium px-2 py-0.5 rounded-full bg-red-100 dark:bg-red-900/30 text-red-700 dark:text-red-300`
Label: "Deactivated"

**Deactivate button (active accounts, not self):**
```
Classes: min-h-12 px-3 py-2 rounded-lg text-sm font-medium
         text-red-600 dark:text-red-400 border border-red-300 dark:border-red-700
         hover:bg-red-50 dark:hover:bg-red-900/20 transition-colors
         disabled:opacity-50 disabled:cursor-not-allowed
Label: "Deactivate"
Loading label: "..."
```

**Reactivate button (deactivated accounts):**
```
Classes: min-h-12 px-3 py-2 rounded-lg text-sm font-medium
         text-accent hover:bg-accent/10 transition-colors
         disabled:opacity-50 disabled:cursor-not-allowed
Label: "Reactivate"
Loading label: "..."
```

**Admin's own row:** Same layout but no action button on the right side. The row is visually identical otherwise.

Source: Directly matches `Settings.tsx` Deleted Sessions row pattern (Restore = accent, Delete Forever = red destructive).

### 5. Deactivation ConfirmDialog

Uses existing `ConfirmDialog` component with `destructive` prop.

| Property | Value |
|----------|-------|
| title | "Deactivate Account" |
| message | "Deactivate {display_name}? They will not be able to log in until reactivated. Their sessions will not be affected." |
| confirmLabel | "Deactivate" |
| cancelLabel | "Cancel" |
| destructive | `true` |

No ConfirmDialog for reactivation (harmless action, per CONTEXT.md).

### 6. Settings Page Admin Section

Added as a new `<section>` in `Settings.tsx`, rendered conditionally when user has admin role. Placed after About section and before Storage section (admin-relevant content first).

**Section header:** `text-sm font-semibold uppercase tracking-wider text-gray-500 dark:text-gray-400 mb-3` with text "Admin".

**Navigation row:**
```
Classes: w-full bg-gray-50 dark:bg-gray-800 rounded-lg p-4 text-left
         text-gray-900 dark:text-gray-100 min-h-12
         hover:bg-gray-100 dark:hover:bg-gray-700 transition-colors
         flex items-center justify-between
```
Left text: "Account Management"
Right: Chevron-right icon (Heroicons, 20px, `text-gray-400`).

Tapping navigates to `/admin/accounts`.

## Interaction States

### Loading States

| Scenario | Visual Treatment |
|----------|-----------------|
| Account list loading | Skeleton: 3 rows of `bg-gray-50 dark:bg-gray-800 rounded-lg p-4 h-20 animate-pulse` with `space-y-3` |
| Creating account (submit button) | Button text changes to "Creating...", button is `disabled` |
| Deactivating account (button) | Button text changes to "...", button is `disabled` |
| Reactivating account (button) | Button text changes to "...", button is `disabled` |

### Error States

| Scenario | Visual Treatment |
|----------|-----------------|
| Account creation fails | Red error text below form fields, above submit button: `text-sm text-red-600 dark:text-red-400 mt-2` |
| Deactivation/reactivation fails | Inline error text below the affected account row: `text-sm text-red-600 dark:text-red-400 mt-1 px-4` (appears temporarily, auto-clears after 5 seconds) |
| Network error loading account list | Full-width error card replacing list: `bg-red-50 dark:bg-red-900/20 rounded-lg p-4 text-red-700 dark:text-red-300 text-sm` with retry button |

### Optimistic Updates

After successful deactivation/reactivation, update the local account list state immediately (toggle `is_active` and swap button). Do not wait for a re-fetch. Re-fetch in background to confirm server state.

After successful account creation, re-fetch the full account list (new account needs server-generated fields like `id` and `created_at`).

## Responsive Behavior

This phase follows the established portrait/landscape pattern from `Settings.tsx`:

| Breakpoint | Behavior |
|------------|----------|
| Portrait (default) | `px-4` horizontal padding, full-width layout |
| Landscape | `px-8 max-w-3xl mx-auto` -- wider padding, centered constrained width |

The account row layout (`flex items-center justify-between`) gracefully handles both orientations. On narrow portrait screens, the display name and email truncate (`truncate` class on both), while the action button remains visible on the right.

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Touch targets | All buttons use `min-h-12` (48px) per UX-02 requirement |
| Form labels | Each input has a visible `<label>` element with `htmlFor` binding |
| Button labels | Deactivate/Reactivate buttons include `aria-label` with account name: `aria-label="Deactivate {display_name}"` |
| ConfirmDialog focus | Focus traps inside dialog (existing ConfirmDialog behavior) |
| Color contrast | All text colors meet WCAG AA against their backgrounds (verified: gray-900 on white, gray-100 on gray-800, red-600 on white, red-400 on gray-800) |
| Loading states | Disabled buttons use `disabled` attribute (not just visual styling) so screen readers announce correctly |
| Error messages | Form error uses `role="alert"` for screen reader announcement |
| Badge semantics | Badges are visual-only (no interactive role); role and status are also conveyed by their text content |
